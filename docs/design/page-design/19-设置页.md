## 4. 交互流程

### 4.1 页面加载

```
用户从个人中心点击「设置」进入
  → 渲染设置页面（静态 TDCellGroup 列表，无需接口请求）
  → 后台静默请求 GET /health 获取后端版本号
  → 版本号返回后缓存到本地（供关于 TDDialog 使用）
```

### 4.2 修改密码

```
用户点击「修改密码」TDCell
  → TDesign 内置点击态反馈
  → 跳转至修改密码页 /mine/change-password
```

### 4.3 退出登录

```
用户点击「退出登录」TDCell
  → 弹出 TDDialog 确认 "确定退出登录？"
  → 用户点击「取消」→ 关闭 TDDialog，无操作
  → 用户点击「确定」：
      → TDDialog 按钮显示 TDLoading（可选）
      → POST /api/auth/logout，请求体携带 { refreshToken }
        （后端强制校验 refreshToken 并吊销对应 jti，缺失返回 VALIDATION）
      → 成功响应：
          → 清除本地 Token（accessToken + refreshToken）和用户缓存
          → 关闭 TDDialog
          → 跳转至登录页（清除任务栈）
      → 失败响应：
          → 仍然清除本地 Token（确保退出）
          → 跳转至登录页
      → 网络错误：
          → 仍然清除本地 Token
          → 跳转至登录页（离线也能退出）
```

### 4.4 全设备退出登录

```
用户点击「全设备退出登录」TDCell
  → 弹出 TDDialog 确认 "确定在所有设备上退出登录？..."
  → 用户点击「取消」→ 关闭 TDDialog
  → 用户点击「全部退出」：
      → POST /api/auth/logout-all
      → 成功响应：
          → 清除本地 Token 和用户缓存
          → 关闭 TDDialog
          → `TDToast` "已在所有设备上退出登录"
          → 跳转至登录页
      → 失败响应：
          → `TDToast` "操作失败，请重试"
          → 不跳转（保留登录态）
```

### 4.5 关于

```
用户点击「关于」TDCell
  → TDesign 内置点击态反馈
  → 弹出关于 TDDialog
  → 展示 APP 版本、后端版本、构建时间
  → 用户点击「确定」→ 关闭 TDDialog
```

## 5. 状态设计

### 5.1 初始态

| 元素 | 表现 |
|------|------|
| TDCellGroup 列表 | 正常渲染（静态配置，无需加载） |
| 版本号 | 后端版本加载中显示 "加载中..."，APP 版本从本地读取即时显示 |
| 底部版本信息 | 后端版本返回后更新 |

### 5.2 加载态

| 场景 | 表现 |
|------|------|
| /health 接口加载中 | 关于 TDDialog 中后端版本显示 "加载中..." |
| 退出登录请求中 | TDDialog 确认按钮可显示 `TDLoading` 指示器 |

### 5.3 错误态

| 场景 | 表现 |
|------|------|
| /health 接口失败 | 后端版本显示 "获取失败"，不影响其他功能 |
| 退出登录接口失败 | 仍然执行本地退出（清 Token + 跳转登录页） |
| 全设备登出接口失败 | `TDToast` "操作失败，请重试"，不执行本地退出 |

### 5.4 成功态

| 场景 | 表现 |
|------|------|
| 退出登录成功 | 跳转登录页，无额外 `TDToast` |
| 全设备登出成功 | `TDToast` "已在所有设备上退出登录"，跳转登录页 |

## 6. 接口依赖

### 6.1 退出登录

> 响应信封等统一遵循 [00-全局API约定](./00-全局API约定.md)。

```
POST /api/auth/logout

请求头：
  Authorization: Bearer {accessToken}

请求体（JSON，必传 refreshToken）：
{
  "refreshToken": "string"     // 当前会话的 refresh token，后端校验后吊销其 jti
}

成功响应（200，{success, data, error} 信封）：
{
  "success": true,
  "data": { "success": true },
  "error": null
}

前端处理：
- 清除本地 Token（accessToken + refreshToken，SharedPreferences / SecureStorage）
- 清除用户缓存信息
- 跳转至登录页
- 注意：即使接口失败，也应清除本地状态并跳转
```

### 6.2 全设备退出登录

```
POST /api/auth/logout-all

请求头：
  Authorization: Bearer {accessToken}

说明：后端自增 tokenVersion，使该账号所有设备的旧 token 失效。

成功响应（200，{success, data, error} 信封）：
{
  "success": true,
  "data": { "success": true },
  "error": null
}

前端处理：
- 仅在后端确认成功后才清除本地状态
- 接口失败时保留当前登录态
```

### 6.3 健康检查（获取后端版本）

```
GET /health

无需鉴权

成功响应（200，{success, data, error} 信封，核验自 src/app.ts）：
{
  "success": true,
  "data": { "ok": true, "ts": 1721635200000 },
  "error": null
}

前端处理：
- 请求失败时显示 "获取失败"

⚠️ 已知缺口：后端 /health 当前仅返回 { ok, ts }，**不返回 version / buildTime
字段**。"后端版本" 展示功能依赖后端扩展 /health 响应（需开后端需求单）；
后端未扩展前，关于 TDDialog 与底部版本信息的后端版本一行显示 "获取失败"。
```

## 7. 边界情况与备注

| 场景 | 处理方式 |
|------|---------|
| /health 接口超时 | 5s 超时，超时后版本显示 "获取失败" |
| 后端版本号格式异常 | 原样显示，不做格式校验 |
| 退出登录时网络断开 | 仍然执行本地退出，跳转登录页 |
| 全设备登出时网络断开 | `TDToast` "网络连接失败"，不执行本地退出 |
| 快速连续点击退出 | TDDialog 关闭后按钮不可再次触发（防重复） |
| 全设备登出后当前设备 | 当前设备也会被登出，跳转登录页 |
| 关于 TDDialog 中版本信息 | APP 版本来自本地 BuildConfig，后端版本来自 /health 接口 |
| 页面返回 | 返回个人中心，无需刷新 |
| 底部版本号更新 | 每次进入设置页重新请求 /health，更新版本号 |
| 全设备登出为可选功能 | 若后端未实现 logout-all 接口，隐藏该 TDCell 列表项 |
| 多租户场景 | **多租户切换 MVP 不支持**（主文档 §1.4 明确不做跨租户），退出登录后需使用其他租户账号重新登录 |
