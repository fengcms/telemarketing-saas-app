## 1. 页面概述

### 1.1 页面定位

跟进编辑弹窗是一个轻量级 `TDDialog`，用于修改已有的跟进备注内容。弹窗从跟进时间线中的编辑按钮触发，仅允许编辑 `content`（跟进备注）字段，不可修改接听类型、通话时长、分类等其它字段。

### 1.2 目标用户

| 角色 | 使用场景 | 限制 |
|------|---------|------|
| **TE（租户员工）** | 修正自己刚提交的备注错别字或内容 | 仅自己的记录 + 创建后 5 分钟内 |
| **TM（租户经理）** | 修正团队成员的备注内容 | 所有记录，无时间限制 |
| **TA（租户管理员）** | 修正任何人的备注内容 | 所有记录，无时间限制 |

### 1.3 核心目标

- **精准修改**：仅允许修改备注内容，防止误改其它字段
- **安全可控**：权限校验 + 时限校验，防止越权编辑
- **快速完成**：弹窗轻量，预填内容，改完即走

### 3.1 TDDialog 容器

| 属性 | 值 |
|------|-----|
| 类型 | `TDDialog`（自定义内容布局） |
| 背景色 | `gray-13`（#FFFFFF 亮色 / `gray-11` #2C2C2C 暗色） |
| 圆角 | `radiusLarge`（12px） |
| 阴影 | `shadowLarge`（offset(0,8), blur 24, gray-3 @30%） |

| 点击外部（遮罩）关闭 | **允许**；若有未保存修改 → 弹出放弃确认（§3.6）；无修改则直接关闭 |

## 4. 交互流程

### 4.1 主流程：编辑并保存

```
用户操作                          系统响应
─────────                        ─────────
1. 在跟进时间线中点击 [✏️] 编辑图标
                                  → 前端权限校验：
                                    TE: followup.userId === currentUser.id
                                        && now - followup.createdAt ≤ 5min
                                    TM/TA: 通过
                                  
                                  → 校验通过：
                                    a) 弹出 TDDialog
                                    b) 标题显示 "编辑跟进备注"
                                    c) TDTextarea 预填当前 content
                                    d) 字数统计显示 "{length}/2000"
                                    e) 保存按钮禁用（内容无修改）
                                    f) 光标定位到文本末尾
                                    g) 弹出软键盘

2. 用户修改文本内容               
   输入/删除/粘贴文字              
                                  → 实时更新字数统计
                                  → 内容与原始不同时：
                                    保存按钮启用（颜色从灰变为主色调）
                                  → 内容达到 2000 字：
                                    字数统计变红（error-7），禁止继续输入
                                  → 内容清空为 0 字：
                                    保存按钮仍可用（允许清空备注）

3. 用户点击 [保存]                 
                                  → 保存按钮进入 loading 态
                                  → 取消按钮禁用
                                  → API: PATCH /api/tenant/leads/:id/followups/:fid
                                    Body: { "content": "修改后的内容" }
                                  
                                  → 成功 (200)：
                                    a) 关闭弹窗（缩放 + 淡出动画 150ms）
                                    b) 收起软键盘
                                    c) TDToast "已保存"（底部居中，2s 消失）
                                    d) 刷新时间线中该条记录（局部更新）
                                    e) 该记录卡片高亮 1s（brand-7 8% → 透明）

4. 弹窗关闭，回到时间线           
                                  → 时间线保持当前滚动位置
                                  → 被编辑的记录显示更新后的内容
```

### 4.2 取消流程

> **关闭途径**：仅"取消"按钮或点击遮罩可关闭弹窗；**物理返回键不关闭弹窗**（保持编辑状态，防误触丢失内容）。

```
场景 A：内容无修改时取消
──────────────────────
用户点击 [取消] 或遮罩            
                                  → 直接关闭弹窗
                                  → 不弹出确认（因为没有未保存修改）

场景 B：内容有修改时取消
──────────────────────
用户点击 [取消] 或遮罩            
                                  → 检测到内容已修改
                                  → 弹出放弃确认 TDDialog（§3.6）

  点击 [继续编辑]                 
                                  → 关闭确认弹窗
                                  → 返回编辑弹窗，内容保留

  点击 [放弃]                     
                                  → 关闭确认弹窗
                                  → 关闭编辑弹窗
                                  → 不保存修改
                                  → 时间线不变

  按返回键（确认弹窗内）          
                                  → 等同于点击 [继续编辑]
                                  → 关闭确认弹窗，回到编辑态

场景 C：编辑弹窗内按物理返回键
──────────────────────
用户按物理/系统返回键             
                                  → 弹窗**不关闭**，保持编辑状态（防误触丢失内容）
                                  → 不做任何提示（或轻量 TDToast "编辑中，可点击取消退出"，MVP 可不加）
```

### 4.3 异常流程

```
场景 A：保存时超过 5 分钟时限（TE）
──────────────────────────────
用户点击 [保存]                   
                                  → API 返回 403：
                                    { "success": false, "data": null,
                                      "error": { "code": "AUTH_FORBIDDEN", "message": "edit window expired (5 min)" } }
                                  → 关闭 loading
                                  → 弹窗内显示错误提示条（顶部）：
                                    ┌───────────────────────────┐
                                    │ ⚠ 编辑时限已过，无法修改   │
                                    └───────────────────────────┘
                                    背景：#FDECEE（浅 error 背景）
                                    文字：#4A1D1A（深 error 文字）
                                    高度：40px，fontBodyMedium (14sp)
                                  → 保存按钮恢复可点击（用户可重试）
                                  → 但重试仍会失败
                                  → 建议：3s 后自动关闭弹窗，返回列表

场景 B：保存时提示非自己的记录
──────────────────────────────
用户点击 [保存]                   
                                  → API 返回 403：
                                    { "success": false, "data": null,
                                      "error": { "code": "AUTH_FORBIDDEN", "message": "not your followup" } }
                                  → 处理方式同场景 A
                                  → 提示文字："没有编辑权限"

场景 C：网络错误
────────────────
用户点击 [保存]                   
                                  → 网络超时或无网络
                                  → 自动重试 1 次（间隔 1s）
                                  → 重试仍失败：
                                    提示条："保存失败，请检查网络"
                                    保存按钮恢复可点击

场景 D：服务器错误 (500)
──────────────────────
用户点击 [保存]                   
                                  → API 返回 500
                                  → 提示条："服务器异常，请稍后重试"
                                  → 保存按钮恢复可点击

场景 E：弹窗打开期间记录被他人删除
──────────────────────────────
                                  → 保存时 API 返回 404
                                  → 提示条："该记录已被删除"
                                  → 2s 后自动关闭弹窗
                                  → 刷新时间线（该记录消失）
```

```
│  禁用态   │ ← 初始态（内容无变化）
└────┬─────┘
     │ 用户修改内容
     ▼
┌──────────┐
│  启用态   │ ← 内容已修改
└──┬───┬───┘
   │   │
   │   │ 点击保存
   │   ▼
   │ ┌──────────┐
   │ │ Loading态 │ ← 请求中
   │ └──┬───┬───┘
   │    │   │
   │ 成功│   │失败
   │    ▼   ▼
   │  弹窗   ┌──────────┐
   │  关闭   │  错误态   │ → 用户可重试
   │         └────┬─────┘
   │              │ 用户修改内容
   │              │
   └──────────────┘ ← 回到启用态

禁用条件汇总：
1. 内容与原始 content 完全一致（trim 后比较）
2. 正在 loading（保存中）
3. 内容超过 2000 字（理论上不会出现，因为禁止输入）
```

## 6. 接口依赖

### 6.1 编辑跟进备注

> 响应信封与错误码以 [00-全局API约定](./00-全局API约定.md) §1/§8 为准。

```
PATCH /api/tenant/leads/:leadId/followups/:fid
Content-Type: application/json
Authorization: Bearer {token}

路径参数：
  leadId : string (UUID) — 线索 ID
  fid    : string (UUID) — 跟进记录 ID

请求体：
{
  "content": "修正后的备注内容"
}

字段约束：
  - 后端仅读取 content 字段（≤ 2000 字符）
  - 请求体中的其它字段（answerType / duration / categoryId 等）会被后端忽略
    （这些字段通过通话记录补正流程处理，见 16-通话记录）

成功响应 (200)：
{
  "success": true,
  "data": { "id": "uuid", "updated": true },
  "error": null
}

错误响应（信封格式）：

400 Bad Request — 参数校验失败：
{
  "success": false, "data": null,
  "error": { "code": "VALIDATION", "message": "String must contain at most 2000 character(s)" }
}

403 Forbidden — 时限超时（TE）：
{
  "success": false, "data": null,
  "error": { "code": "AUTH_FORBIDDEN", "message": "edit window expired (5 min)" }
}

403 Forbidden — 非自己的记录（TE）：
{
  "success": false, "data": null,
  "error": { "code": "AUTH_FORBIDDEN", "message": "not your followup" }
}

404 Not Found：
{
  "success": false, "data": null,
  "error": { "code": "NOT_FOUND", "message": "followup not found" }
}
```

### 6.2 辅助数据（弹窗打开前已获取）

| 数据 | 来源 | 说明 |
|------|------|------|
| 跟进记录原始内容 | `GET /api/tenant/leads/:id/followups` 列表返回 | 弹窗打开时从列表缓存中读取，无需额外请求 |
| 当前用户角色 | 登录时缓存在本地 profile | 用于前端权限判断 |
| 当前用户 ID | 同上 | 用于判断是否为自己的记录 |

## 7. 边界情况与备注

### 7.1 权限边界

| 场景 | 前端处理 | 后端校验 |
|------|---------|---------|
| TE 点击他人记录的编辑图标 | 编辑按钮不显示（时间线侧控制）；若通过其它入口触发 → `TDToast` "没有编辑权限" | 403 "没有编辑权限" |
| TE 编辑超 5 分钟 | 编辑按钮不显示；若弹窗已打开超时 → 保存时 403 → 提示 | 403 "编辑时限已过" |
| TE 编辑自己的记录（≤ 5min） | 正常打开弹窗，正常保存 | 200 成功 |
| TM/TA 编辑任何记录 | 正常打开，正常保存 | 200 成功 |
| 弹窗打开期间超过 5 分钟 | 弹窗保持打开；保存时后端返回 403 → 提示 | 403 |

### 7.2 数据边界

| 场景 | 处理 |
|------|------|
| content 为空字符串 | 允许保存（清空备注是合法操作） |
| content 为纯空格 | trim 后为空 → 视为空字符串，允许保存 |
| content 含换行符 | 保留换行，正常提交 |
| content 含特殊字符（HTML/SQL） | 前端不限制；后端做 XSS 过滤和 SQL 参数化 |
| content 超过 2000 字 | 前端禁止输入超过 2000 字；后端也应校验 max length |
| content 与原始完全一致 | 保存按钮禁用，不发送请求 |
| content 仅空格差异 | trim 后比较；若 trim 后一致 → 保存按钮禁用 |
| content 含 emoji | 正常输入和保存；1 个 emoji 计为 1 个字符 |

### 7.3 并发与竞态

| 场景 | 处理 |
|------|------|
| 快速连续点击保存 | 第一次点击后按钮立即 loading + 禁用，防止重复提交 |
| 弹窗打开期间记录被他人删除 | 保存时 404 → 提示"该记录已被删除" → 自动关闭弹窗 → 刷新时间线 |
| 弹窗打开期间记录被他人编辑 | 后端以最后写入为准（last-write-wins）；前端保存成功后刷新显示最新内容 |
| 软键盘弹出遮挡保存按钮 | 弹窗内容用 `SingleChildScrollView` 包裹，配合 `MediaQuery.viewInsets.bottom` 调整弹窗位置/内边距，确保文本域与按钮区不被键盘遮挡（`resizeToAvoidBottomInset` 是 Scaffold 属性，TDDialog 不适用） |

### 7.4 输入法与焦点

| 项目 | 要求 |
|------|------|
| 自动弹出键盘 | 弹窗打开后立即弹出软键盘 |
| 光标位置 | 定位到文本末尾（`controller.selection = TextSelection.collapsed(offset: content.length)`） |
| 键盘类型 | 默认文本键盘（`TextInputType.multiline`） |
| 回车键行为 | 换行（不提交） |
| 键盘收起 | 保存成功/取消后自动收起 |
| 键盘遮挡 | 弹窗内容 `SingleChildScrollView` + `MediaQuery.viewInsets.bottom` 自适应，确保文本域与按钮区可见 |
| 粘贴支持 | 支持系统粘贴板；粘贴后检查总字数，超出部分截断 |

### 7.5 无障碍

| 项目 | 要求 |
|------|------|
| 弹窗标题 | `Semantics(header: true)` |
| 输入框 | semanticsLabel = "跟进备注编辑框" |
| 字数统计 | semanticsLabel = "已输入 X 字，最多 2000 字" |
| 保存按钮 | semanticsLabel = "保存修改"；禁用时 "保存修改，当前不可用" |
| 取消按钮 | semanticsLabel = "取消编辑" |
| 错误提示 | semanticsLabel = "错误：编辑时限已过" |
| 焦点管理 | 弹窗打开后焦点自动移到输入框 |

### 7.8 与时间线组件的联动

```
编辑保存成功后的联动流程：

1. TDDialog 编辑弹窗关闭
2. 通知时间线组件刷新该条记录
   → 方式 A：EventBus / StateFlow 通知
   → 方式 B：回调函数 (onFollowupUpdated(followupId))
3. 时间线组件局部刷新该条记录
   → 仅更新 content 字段
   → 不重新请求整个列表（可选优化）
   → 或：重新请求该页数据（简单实现）
4. 被编辑的记录卡片高亮 1s
5. TDToast "已保存" 显示 2s
```

### 7.9 与通话结果反馈面板的关系

本文档的编辑弹窗与 `07-通话结果反馈面板.md` 是**独立**的两个组件：

| 维度 | 通话结果反馈面板 | 跟进编辑弹窗 |
