## 1. 页面概述

| 项目 | 说明 |
|------|------|
| **页面定位** | 展示用户的通话记录列表及详情，支持按日期和接听类型筛选 |
| **目标用户** | 所有已登录用户（TE / TM / TA） |
| **核心目标** | 让用户查看自己的通话记录详情，TM/TA 可查看全部记录并进行编辑/删除操作 |
| **页面路由** | `/mine/call-records` |
| **入口** | 个人中心 → 通话记录 |
| **数据范围** | TE：仅自己的通话记录；TM/TA：全部通话记录 |

## 4. 交互流程

### 4.1 列表页加载

```
用户从个人中心点击「通话记录」进入
  → 显示 TDNavBar + 筛选区域 + TDSkeleton 列表骨架（5 条）
  → GET /api/tenant/calls?dateFrom={30天前}&dateTo={今天}&sort=-startedAt&page=1&size=20
  → 数据返回后渲染通话记录 TDCell 列表
  → 隐藏 TDSkeleton
```

### 4.2 下拉刷新

```
用户在列表顶部下拉
  → 显示 TDRefreshHeader 刷新指示器
  → 重新请求 GET /api/tenant/calls（保持当前筛选条件，page=1&size=20）
  → 数据返回后刷新列表并重置分页状态
  → 刷新结束，收起 TDRefreshHeader
```

### 4.3 日期范围筛选

```
用户点击日期区域
  → 弹出 TDDialog + TDCalendarPicker
  → 默认显示当前日期范围
  → 用户选择新范围 → 点击「确定」
  → 列表区域显示 TDSkeleton
  → 重新请求（携带新 dateFrom/dateTo，page 重置为 1）
  → 刷新列表
```

### 4.4 接听类型筛选

```
用户点击 TDTag（如「已接听」）
  → TDTag 切换选中态（原选中 Tag 取消，新 Tag 选中）
  → 列表区域显示 TDSkeleton
  → 重新请求（携带 answerType 参数，page 重置为 1）
  → 刷新列表
  → 点击「全部」时不传 answerType 参数
```

### 4.5 无限滚动

```
用户滚动到列表底部（距底部 < 200px）
  → 底部显示 TDLoading（居中，`brand-7`）
  → GET /api/tenant/calls?...&page={当前页+1}&size=20
  → 数据追加到列表末尾
  → 若无更多数据（page >= pages）→ 底部显示 "没有更多了"（`fontBodyExtraSmall`，`gray-6`，居中，上间距 `spacer16`）
```

### 4.6 点击通话记录 → 详情

```
用户点击 TDCell 列表项
  → TDesign 内置点击态反馈
  → 跳转至通话详情页 /mine/call-records/:id
  → GET /api/tenant/calls/:id
  → 渲染详情
```

### 4.7 编辑通话记录（TM/TA）

```
TM/TA 用户在详情页点击「编辑」
  → 弹出 TDDialog 编辑弹窗
  → 弹窗预填当前值（接听类型 + 时长 + 结束时间）
  → 用户修改后点击「保存」
  → PATCH /api/tenant/calls/:id { answerType, duration, endedAt }
    （三字段均为可选，仅传被修改的字段；endedAt 为 int Unix 秒）
  → 成功 → 关闭 TDDialog → 刷新详情页
  → 失败 → TDDialog 内显示错误提示
```

### 4.8 删除通话记录（TA）

```
TA 用户在详情页点击「删除通话记录」
  → 弹出 TDDialog 确认 "确定删除此通话记录？"
  → 确认按钮 `TDButton(theme: TDButtonTheme.danger)` "删除" / 取消按钮 `TDButton(theme: TDButtonTheme.text)` "取消"
  → 确认删除 → 软删除请求
  → 成功 → `TDToast` "删除成功" → 返回列表页
  → 失败 → `TDToast` "删除失败，请重试"
```

## 6. 接口依赖

### 6.1 获取通话记录列表

> 响应信封、分页、排序、枚举等统一遵循 [00-全局API约定](./00-全局API约定.md)。

```
GET /api/tenant/calls

请求参数：
| 参数       | 类型   | 必填 | 说明                           |
|------------|--------|------|-------------------------------|
| dateFrom   | string | 否   | 起始日期 YYYY-MM-DD            |
| dateTo     | string | 否   | 结束日期 YYYY-MM-DD            |
| answerType | string | 否   | 接听类型筛选（五值枚举，见下）  |
| sort       | string | 否   | 排序，默认 sort=-startedAt（- 前缀为降序） |
| page       | int    | 否   | 页码，从 1 起，默认 1           |
| size       | int    | 否   | 每页条数，默认 20，最大 200     |
| 其他       | —      | 否   | 支持通用查询 DSL（驼峰字段名）  |

answerType 取值：answered / no_answer / rejected / empty_number / suspended
（"外呼"属于 direction 维度，不作为 answerType 参数）

数据范围：
- TE：自动限定为当前用户记录（后端根据 Token 判断）
- TM/TA：返回全部记录

成功响应（{success, data, error} 信封，分页结构 {items,total,page,size,pages}）：
{
  "success": true,
  "data": {
    "items": [
      {
        "id": "call_001",
        "calleeNumber": "13812341234",    // 被叫号码（明文）
        "startedAt": 1705300225,          // int Unix 秒
        "duration": 205,                  // 秒
        "answerType": "answered",         // answered/no_answer/rejected/empty_number/suspended
        "violation": 0,                   // 0=无, 1=有
        "leadId": "lead_001",             // 关联线索 ID
        "leadName": "张先生"              // 关联线索姓名
      }
    ],
    "total": 128,
    "page": 1,
    "size": 20,
    "pages": 7
  },
  "error": null
}
```

```
请求体（三字段均可选，仅传需要补正的字段）：

  "answerType": "answered",   // 可选，五值枚举
  "duration": 205,            // 可选，单位秒
  "endedAt": 1705300430       // 可选，结束时间，int Unix 秒
}

成功响应（{success, data, error} 信封）：
{
  "success": true,
  "data": { "id": "call_001", "updated": ["answerType", "duration", "endedAt"] },
  "error": null
}

权限：仅 TM/TA（后端 requireManager 校验，TE 调用返回 403 AUTH_FORBIDDEN）
```

## 7. 边界情况与备注

| 场景 | 处理方式 |
|------|---------|
| 通话时长为 0 | 显示 "0 秒"；无人接听/拒接/空号/停机类型显示 "未接通" |
| 下拉刷新 | 列表页支持 `TDRefreshHeader` 下拉刷新，保持筛选条件并重置分页（page=1） |
| 被叫号码为空 | 显示 "未知号码" |
| 无关联线索 | 详情页关联线索区域显示 "无关联线索" |
| 关联线索已被删除 | 显示 "线索已删除"（`gray-5`，不可点击） |
| 违规标记 violation=1 | TDCell 右侧显示 `TDIcons.error_circle` `error-7` 图标，详情页违规字段显示 `error-7` "有" |
| TE 尝试编辑/删除 | 前端隐藏编辑/删除按钮，后端也会做权限校验 |
| 日期范围超过 1 年 | 允许，但提示 "查询范围较大，加载可能较慢" |
| 列表快速滚动 | 使用 ListView 复用机制，保证流畅 |
| 详情页返回 | 列表页保持滚动位置和筛选状态 |
| 编辑 TDDialog 中取消 | 丢弃修改，关闭 TDDialog，不发送请求 |
| duration 输入非法值 | 前端校验：必须为非负整数，否则 TDInput 显示 `error-7` 边框 |
