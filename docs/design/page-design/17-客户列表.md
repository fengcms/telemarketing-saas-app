# 17 - 客户列表

> ⚠️ **版本规划说明（产品决断）**：客户管理模块（本页及 [18-客户详情](./18-客户详情.md)）为 **v1.1 版本规划**，**MVP（v1.0）不含客户管理，仅线索管理**。本文档作为 v1.1 设计预留，MVP 阶段不开发、不验收。

> **设计规范参考**：[00-TDesign-Flutter-设计规范](./00-TDesign-Flutter-设计规范.md)
> 组件库：[TDesign Flutter](https://github.com/Tencent/tdesign-flutter)
> 色彩 / 字体 / 间距 / 圆角 / 阴影 / 组件均遵循上述设计规范。

---

## 1. 页面概述

| 项目 | 说明 |
|------|------|
| **页面定位** | 展示用户已成功转化的客户列表，支持搜索和筛选 |
| **目标用户** | 所有已登录用户（TE / TM / TA） |
| **核心目标** | 让用户查看和管理自己的客户资源，TE 仅看自己的客户，TM/TA 可切换查看全部 |
| **页面路由** | `/mine/customers` |
| **入口** | 个人中心 → 客户列表 |
| **数据范围** | TE：scope=mine（仅自己的客户）；TM/TA：可切换全部/个人 |
| **关键约束** | 所有请求均携带 `erased=0` 参数，排除已删除客户 |

## 4. 交互流程

### 4.1 页面加载

```
用户从个人中心点击「客户列表」进入
  → 显示 TDNavBar + TDSearchBar + TDTag 行 + TDSkeleton 列表骨架（5 条）
  → GET /api/tenant/customers?erased=0&scope=mine&sort=-convertedAt&page=1&size=20
  → 数据返回后渲染客户卡片列表
  → 隐藏 TDSkeleton
```

### 4.2 下拉刷新

```
用户在列表顶部下拉
  → 显示 TDRefreshHeader 刷新指示器
  → 重新请求 GET /api/tenant/customers（保持当前搜索/筛选条件，page=1&size=20）
  → 数据返回后刷新列表并重置分页状态
  → 刷新结束，收起 TDRefreshHeader
```

### 4.3 Scope 切换（TM/TA）

```
TM/TA 用户点击 TDNavBar 右侧 scope 切换
  → 弹出 TDPicker 弹出菜单：「我的客户」/「全部客户」
  → 选择后：
    → 列表区域显示 TDSkeleton
    → 重新请求（scope=mine 或 scope=all，page 重置为 1）
    → 刷新列表
    → TDNavBar 右侧文字更新
```

### 4.4 搜索

```
用户在 TDSearchBar 输入关键词
  → 输入停止 500ms 后触发防抖搜索
  → GET /api/tenant/customers?erased=0&scope=mine&keyword={keyword}&...
  → 列表区域显示 TDSkeleton
  → 刷新列表
```

```
用户点击 TDSearchBar 清除按钮
  → 清空搜索框
  → 重新加载无 keyword 的列表
```

### 4.5 等级筛选

```
用户点击 TDTag（如「A级」）
  → TDTag 切换选中态
  → GET /api/tenant/customers?erased=0&scope=mine&level=A&...
  → 列表区域显示 TDSkeleton
  → 刷新列表
```

### 4.6 无限滚动

```
用户滚动到列表底部（距底部 < 200px）
  → 底部显示 TDLoading
  → GET /api/tenant/customers?...&page={当前页+1}&size=20
  → 数据追加到列表末尾
  → 若无更多数据（page >= pages）→ 底部显示 "没有更多了"
```

### 4.7 点击客户卡片

```
用户点击客户卡片
  → TDesign 内置点击态反馈
  → 跳转至客户详情页 /mine/customers/:id
```

## 5. 状态设计

### 5.1 加载态

| 区域 | 表现 |
|------|------|
| 初始加载 | `TDSkeleton`：5 条客户卡片骨架（姓名骨架 + 标签骨架 + 3 行文字骨架） |
| 加载更多 | 底部 `TDLoading` |
| 搜索/筛选 | 列表区域 `TDSkeleton` |

### 5.2 空态

| 场景 | 表现 |
|------|------|
| 无客户数据 | `TDEmpty`：人物图标插图 + "暂无客户"（`fontTitleSmall`，`gray-6`，居中） |
| 搜索无结果 | `TDEmpty`："未找到匹配的客户"（`fontTitleSmall`，`gray-6`，居中）+ `TDButton(theme: TDButtonTheme.text)` "清除搜索" |

### 5.3 错误态

| 场景 | 表现 |
|------|------|
| 列表加载失败 | `TDEmpty` 错误插图 + `TDButton(theme: TDButtonTheme.text)` "重试" |
| 网络断开 | `TDToast` "网络连接失败" + 重试操作 |

### 5.4 成功态

数据正常渲染，无额外成功提示。

## 6. 接口依赖

### 6.1 获取客户列表

> 响应信封、分页、排序等统一遵循 [00-全局API约定](./00-全局API约定.md)。

```
GET /api/tenant/customers

请求参数：
| 参数     | 类型   | 必填 | 说明                          |
|----------|--------|------|------------------------------|
| erased   | int    | 是   | 固定传 0，排除已删除          |
| scope    | string | 否   | mine=我的客户，all=全部（TM/TA）|
| keyword  | string | 否   | 搜索关键词（姓名/电话/公司）   |
| level    | string | 否   | 等级筛选 A/B/C/D              |
| sort     | string | 否   | 排序，默认 sort=-convertedAt（- 前缀为降序） |
| page     | int    | 否   | 页码，从 1 起，默认 1          |
| size     | int    | 否   | 每页条数，默认 20，最大 200    |
| 其他     | —      | 否   | 支持通用查询 DSL（驼峰字段名） |

成功响应（{success, data, error} 信封，分页结构 {items,total,page,size,pages}）：
{
  "success": true,
  "data": {
    "items": [
      {
        "id": "cust_001",
        "name": "张*",                   // 脱敏姓名
        "phone": "138****1234",          // 脱敏电话
        "company": "XX科技有限公司",
        "level": "A",                    // A/B/C/D
        "convertedAt": "2024-01-10",     // 转化日期
        "leadId": "lead_001"             // 来源线索 ID
      }
    ],
    "total": 36,
    "page": 1,
    "size": 20,
    "pages": 2
  },
  "error": null
}
```

## 7. 边界情况与备注

| 场景 | 处理方式 |
|------|---------|
| 客户姓名为空 | 显示 "未命名客户" |
| 电话为空 | 显示 "无联系电话" |
| 公司为空 | 隐藏公司行 |
| 转化日期为空 | 显示 "转化日期: —" |
| 客户等级为空 | 不显示 TDTag 等级标签 |
| 搜索关键词包含特殊字符 | 前端 URL encode 后传给后端 |
| 快速输入搜索 | 500ms 防抖（与主文档 §4.4.1 一致），取消前一次请求 |
| 下拉刷新 | 支持 `TDRefreshHeader` 下拉刷新，保持搜索/筛选条件并重置分页（page=1） |
| TE 尝试切换 scope | 前端隐藏 scope 切换按钮，后端也做权限校验 |
| 列表返回 | 保持搜索、筛选、滚动位置状态 |
| 客户数据量极大 | 使用 ListView + TDLoadMore 分页加载，保证性能 |
| 客户已被删除（erased=1） | 不展示（请求已携带 erased=0） |
