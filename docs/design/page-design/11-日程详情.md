## 1. 页面概述

| 项目 | 说明 |
|------|------|
| **页面定位** | 日程详情页，展示单条日程的完整信息，并提供基于当前状态的操作按钮 |
| **目标用户** | TE（电销员）、TM（电销主管）、TA（电销管理员） |
| **核心目标** | 1. 查看日程完整信息（标题、内容、关联线索、时间等）；2. 根据日程状态执行对应操作（完成/取消/编辑/重新打开/删除/拨号）；3. 快速跳转关联线索详情 |
| **入口** | 从日程列表页点击某条日程卡片进入 |
| **权限说明** | 查看：本人可查看自己的日程，TM/TA 可查看团队内任意日程（完整权限矩阵见 §3.9）；编辑（title/content/scheduledAt）：日程创建者、TM、TA 可编辑，TE 不可编辑他人日程；删除：仅归属人可删自己的，TA 可删全部（TM 不可删他人日程） |

## 4. 交互流程

### 4.1 页面初始化

```
用户操作                          系统响应
─────────                        ─────────
1. 从日程列表点击某条目    →    2. 页面跳转，传入 scheduleId
                            →    3. 显示 TDSkeleton 骨架屏（卡片布局 Shimmer）
                            →    4. 请求 GET /api/tenant/schedules/:id
                            →    5. 数据返回后渲染页面内容
                            →    6. 根据 status 和 scheduledAt 计算当前可用操作按钮
                            →    7. TDSkeleton 淡出，内容淡入（200ms）
```

### 4.2 完成操作

```
用户操作                          系统响应
─────────                        ─────────
1. 点击「完成」按钮        →    2. 按钮显示 loading 状态（TDLoading 覆盖）
                            →    3. 请求 POST /api/tenant/schedules/:id/complete
                            →    4. 成功（200）：
                                   a. TDToast "日程已完成"
                                   b. 重新请求详情接口刷新页面
                                   c. 状态标签变为"已完成"
                                   d. 底部操作栏变为 completed 状态的按钮组合
                                   e. 标题文字变灰
                            →    5. 失败（4xx/5xx）：
                                   a. 按钮恢复可用
                                   b. TDToast "操作失败，请重试"
```

### 4.3 取消操作

```
用户操作                          系统响应
─────────                        ─────────
1. 点击「取消」按钮        →    2. 弹出 TDDialog 确认弹窗：
                                   标题："取消日程"
                                   内容："确定要取消该日程吗？取消后可重新打开。"
                                   按钮：[取消操作] [确定取消]
                            →    3. 点击「确定取消」
                            →    4. TDDialog 关闭，按钮显示 loading
                            →    5. 请求 POST /api/tenant/schedules/:id/cancel
                            →    6. 成功：
                                   a. TDToast "日程已取消"
                                   b. 刷新页面数据
                                   c. 状态标签变为"已取消"
                                   d. 底部操作栏变为 cancelled 状态按钮组合
                            →    7. 失败：TDToast "操作失败，请重试"
                            →    8. 点击「取消操作」：关闭弹窗，无其他动作
```

### 4.4 编辑操作（含改期）

> 入口：TDNavBar 右侧 ⋮ 菜单 → "编辑"（仅有编辑权限时显示，见 §3.8/§3.9）。编辑覆盖 title/content/scheduledAt 三个字段，"改期"即仅修改 scheduledAt 的编辑。

```
用户操作                          系统响应
─────────                        ─────────
1. 点击 ⋮ →「编辑」        →    2. 跳转日程编辑页（Push，复用 `12-新建日程.md`
                                   的全屏表单，进入编辑模式）：
```

## 6. 接口依赖

### 6.1 获取日程详情

```
GET /api/tenant/schedules/:id
```

**响应结构**（信封格式见 `00-全局API约定.md`；时间字段为 int Unix 秒，展示层格式化为 `YYYY-MM-DD HH:mm`）：

```json
{
  "success": true,
  "data": {
    "id": "sch_001",
    "tenantId": "t_001",
    "userId": "user_001",
    "leadId": "lead_001",
    "callRecordId": "cr_001",
    "title": "跟进张先生购房意向",
    "content": "客户对三居室感兴趣，需要本周内安排看房",
    "status": "pending",
    "scheduledAt": 1784008800,
    "completedAt": null,
    "createdAt": 1783908000,
    "updatedAt": 1783927800,
    "lead": {
      "name": "张伟",
      "phone": "138****1234"
    },
    "call": {
      "id": "cr_001",
      "answerType": "answered",
      "duration": 128,
      "startedAt": 1783927800
    }
  },
  "error": null
}
```

**字段说明：**

| 字段 | 说明 |
|------|------|
| `userId` | 日程归属人 ID（UUID 字符串）。⚠️ 后端详情接口**不返回**归属人姓名对象，"其他信息卡片-归属人"需前端用 `GET /api/tenant/options/users` 结果本地映射，或待后端补充 join（见 §7.3） |
| `lead` | 关联线索快照 `{name, phone}` 或 `null`（线索已擦除）。phone 为明文（REQ-08 已上线）；暂无 `project`/`id` 字段 |
| `call` | 关联通话记录 `{id, answerType, duration, startedAt}` 或 `null`（无 `callRecordId` 时）；`answerType` 枚举见 00 文档 §4.1 |
| `scheduledAt` / `createdAt` / `updatedAt` / `completedAt` | int Unix 秒（UTC） |

**错误：** 404 `NOT_FOUND`（日程不存在或已删除）。

### 6.2 完成日程

```
POST /api/tenant/schedules/:id/complete
```

| 参数 | 位置 | 说明 |
|------|------|------|
| `id` | path | 日程 ID |

**响应：** `{ "success": true, "data": { "id": "sch_001", "status": "completed" }, "error": null }`

**权限与错误：** TE 仅可完成自己的日程（他人 → 403 `AUTH_FORBIDDEN`）；日程非 pending 状态 → 400 `VALIDATION`。

### 6.3 取消日程

```
POST /api/tenant/schedules/:id/cancel
```

| 参数 | 位置 | 说明 |
|------|------|------|
| `id` | path | 日程 ID |

**响应：** `{ "success": true, "data": { "id": "sch_001", "status": "cancelled" }, "error": null }`

**权限与错误：** TE 仅可取消自己的日程（他人 → 403 `AUTH_FORBIDDEN`）；日程非 pending 状态 → 400 `VALIDATION`。

### 6.4 重新打开日程

```
POST /api/tenant/schedules/:id/reopen
```

| 参数 | 位置 | 说明 |
|------|------|------|
| `id` | path | 日程 ID |

**响应：** `{ "success": true, "data": { "id": "sch_001", "status": "pending" }, "error": null }`

**权限与错误：** TE 仅可重新打开自己的日程（他人 → 403 `AUTH_FORBIDDEN`）；日程已是 pending → 400 `VALIDATION`。

### 6.5 修改日程（编辑：标题/内容/改期）

```
PATCH /api/tenant/schedules/:id
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | path | 是 | 日程 ID |
| `scheduledAt` | int（Unix 秒） | 否 | 新的计划时间（DateTime → Unix 秒换算见 00 文档 §7） |
| `title` | string | 否 | 新的标题，≤ 200 字 |
| `content` | string | 否 | 新的内容，≤ 2000 字 |

> 仅传需要变更的字段；后端不支持通过 PATCH 修改归属人 `userId`（改归属人需删除重建）。

**响应：** `{ "success": true, "data": { "id": "sch_001", "changed": ["scheduledAt", "title"] }, "error": null }`（无任何字段变更时 `changed` 为 `false`）

**权限与错误：** 日程创建者、TM、TA 可编辑；TE 编辑他人日程 → 403 `AUTH_FORBIDDEN`；字段超限 → 400 `VALIDATION`。

### 6.6 删除日程

```
DELETE /api/tenant/schedules/:id
```

| 参数 | 位置 | 说明 |
|------|------|------|
| `id` | path | 日程 ID |

**响应：** `{ "success": true, "data": { "id": "sch_001", "deleted": true }, "error": null }`

**权限：** 仅归属人可删自己的日程，TA 可删所有日程；**TM 不可删他人日程**（403 `AUTH_FORBIDDEN`）。后端为软删除（写入 `deletedAt`）。

## 7. 边界情况与备注

### 7.1 边界情况

| 场景 | 处理方式 |
|------|----------|
| 日程标题为空 | 显示"未命名日程" |
| 日程内容为空 | 内容卡片显示"暂无内容"（`gray-6` 灰色斜体） |
| 关联线索已擦除（`lead` 为 null） | 线索卡片显示"该线索已被删除"，不可点击；拨号按钮不显示 |
| 归属人为空 | 其他信息卡片中归属人行显示"未知" |
| scheduledAt 恰好等于当前时间 | 视为已过期（`scheduledAt < now` 判断改为 `scheduledAt <= now`） |
| 编辑时选择过去的时间 | 允许，保存后显示"已逾期" `TDTag` |
| 编辑未变更任何字段 | 不发送请求（或后端返回 `changed: false`），`TDToast` "内容未变更" |
| 快速连续点击操作按钮 | 按钮立即进入 loading/disabled 状态，防止重复提交 |
| 操作期间网络断开 | loading 结束，`TDToast` "网络异常，请重试" |
| 从列表进入详情后，返回列表时列表状态保持 | 保持滚动位置和筛选状态 |
| 日程在详情页被其他用户删除 | 操作时返回 404，`TDToast` "该日程已被删除"，自动返回列表 |
| TM/TA 查看他人日程 | 正常显示所有信息；⋮ 菜单中编辑项对 TM/TA 显示，删除项仅 TA 显示（权限矩阵见 §3.9） |
| 拨号后用户未实际呼出（取消拨号器） | 返回 APP 后仍弹出反馈面板，由用户选择接听类型（如"无人接听"）或关闭面板，见 07 文档 |
| 已完成/已取消的日程，重新打开后又过期 | 显示"已逾期" `TDTag`，按钮组合为 pending（已过期）状态 |

### 7.2 状态与按钮映射速查表

> 完成/取消/拨号/重新打开在底部操作栏；编辑/删除在 ⋮ 菜单（按 §3.9 权限矩阵显示）。

| 当前状态 | 是否过期 | 完成 | 取消 | 编辑（⋮） | 删除（⋮） | 拨号 | 重新打开 |
|---------|---------|------|------|-----------|-----------|------|---------|
| pending | 否 | ✅ | ✅ | ✅（有权限时） | ✅（有权限时） | ✅（lead 存在时） | - |
| pending | 是 | ✅ | ✅ | ✅（有权限时） | ✅（有权限时） | ✅（lead 存在时） | - |
| completed | - | - | - | ✅（有权限时） | ✅（有权限时） | - | ✅ |
| cancelled | - | - | - | ✅（有权限时） | ✅（有权限时） | - | ✅ |

### 7.3 备注

| 项目 | 说明 |
|------|------|
| 页面转场 | 从列表进入：标准右滑入（300ms，`Curves.easeInOut`）；返回列表：标准右滑出（250ms） |
| 数据刷新策略 | 每次操作成功后重新请求详情接口，确保数据一致性 |
| 暗色模式 | 所有颜色使用 TDesign Token，通过 `TDTheme.needMultiTheme()` 自动适配暗色模式 |
| 无障碍 | 所有按钮有 `semanticsLabel`；卡片支持 TalkBack 朗读 |
| 返回键行为 | 系统返回键等同于点击 TDNavBar 返回箭头，返回日程列表 |
