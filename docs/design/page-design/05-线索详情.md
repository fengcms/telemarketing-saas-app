## 1. 页面概述

### 1.1 页面定位

线索详情页是 TE/TM/TA 对单条线索进行查看、跟进、拨号、编辑等核心操作的工作台。页面整合了线索基础信息、操作按钮组、跟进时间线、通话记录摘要四大功能区域，提供一站式的线索管理能力。

### 1.2 目标用户

| 角色 | 权限差异 |
|------|----------|
| TE（电销坐席） | 查看自己线索详情；拨号、跟进、编辑（仅 `pending→following` 前向流转）；编辑/删除**自己的**跟进记录（编辑仅限内容、创建后 5 分钟内） |
| TM（电销主管） | 查看团队线索详情；全部操作权限；查看归属人信息；编辑任意跟进记录内容；**补正通话记录**（`PATCH /api/tenant/calls/:id` 修改 answerType/duration/endedAt） |
| TA（电销管理员） | 同 TM，且可通过 `?raw=1` 查看明文姓名/电话 |

### 1.3 核心目标

- 完整展示线索基础信息
- 提供便捷的拨号、跟进、预约、编辑操作入口
- 通过时间线回顾完整跟进历史
- 通过通话记录摘要快速了解通话情况

```
                               → 3. 边界检查：
                                  ├─ nextIndex >= ids.length → 按钮已 disabled，不响应
                                  └─ nextIndex < ids.length → 继续
                               → 4. 取 targetId = ids[nextIndex]
                               → 5. 页面切换动画：

                                   （缓存，下次切换时直接使用）

1b. 点击 [← 上一个]            → 同上，方向相反（prevIndex = currentIndex - 1）
                                  动画方向：当前页面向右滑出，新页面从左侧滑入

异常处理：
                               → 目标线索接口返回 404（已删除/已转移）：
                                  ├─ 自动跳过该条，继续到下一条
                                  ├─ TDToast 提示 "该线索已不可用，已自动跳过"
                                  └─ 从 listContext.ids 中移除该 ID，更新计数

                               → 目标线索接口返回 403（权限变更）：

                                  └─ TDToast 提示 "网络异常，请重试"
```

### 6.1 获取线索详情

```json
{

  "data": {

    "name": "张*明",
    "phone": "138****1234",

    "position": "采购经理",
    "status": "following",
    "category": "住宅",
    "project": {
      "id": 5,

    },

    "nextFollowupAt": 1752998400,
    "createdAt": 1752138000,

  },

```

> 时间字段为 Unix 秒级 int 时间戳，展示层格式化为 `YYYY-MM-DD HH:mm`。

### 6.2 获取跟进时间线

```
GET /api/tenant/leads/:id/followups
```

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | int | 路径参数 | 线索 ID |

> ⚠️ 该接口**当前不分页**（不接受 page/size），一次返回该线索全部未删除的跟进记录（按 createdAt 倒序）。§3.4.4「加载更多跟进记录」由前端对 `data.items` 本地切片实现（如先渲染前 10 条，点击加载更多再展示下一批）。

**响应结构（成功 200，全局信封）：**

```json
{
  "success": true,
  "data": {
    "items": [
      {
        "id": "fu-uuid-501",
        "leadId": "10001",
        "userId": "3",
        "content": "客户表示有兴趣，需要再考虑一下",
        "categoryId": "住宅",
        "answerType": "answered",
        "duration": 205,
        "createdAt": 1752841800
      }
    ]
  },
  "error": null
}
```

> **前端适配说明**（与后端现状对齐）：
> - 记录仅返回原始字段，**不返回** `operator` / `canEdit` / `canDelete` / `categoryBefore` / `categoryAfter` / `statusBefore` / `statusAfter`。
> - 「跟进人: {姓名}」：前端用 `userId` 通过 `GET /api/tenant/options/users` 缓存的用户映射解析。
> - 编辑/删除按钮可见性由**前端计算**：TE 需 `userId === 当前用户`（编辑另需 `createdAt` 距今 ≤ 300 秒）；TM/TA 不限。
> - 「分类变更: A → B」展示：后端仅提供本条记录的 `categoryId`，无变更前值，前端可与上一条记录的 `categoryId` 比对近似展示（或不展示，见 §7.2 备注）。

## 7. 边界情况与备注

### 7.1 边界情况

| 场景 | 处理方式 |
|------|----------|
| 线索已被删除 | 详情页接口返回 404 → `TDEmpty` 显示 "该线索已被删除" + `TDButton` 返回按钮 |
| 线索已被擦除 | 同已删除处理 |
| 线索已被转化 | 显示详情页但操作区隐藏拨号/跟进按钮，显示 "已转化" `TDTag` 状态 |
| 电话号码为空 | 电话号码区域显示 "暂无电话"，`TDFloatingActionButton` disabled |
| 跟进内容为空提交 | `TDTextarea` 下方显示红色 `error-7` 提示 "请输入跟进内容" |
| 跟进内容达到 2000 字 | 禁止继续输入，计数器显示 `error-7` 色 |
| 通话时长为 0 但选择已接听 | 允许提交，不强制校验时长 |
| TE 编辑跟进超过 5 分钟 | 编辑按钮隐藏，仅保留删除按钮（TM/TA 不受 5 分钟限制） |
| TE 尝试编辑状态到非前向状态 | 状态 TDPicker 中不显示不可达状态 |
| 折叠区域快速滑动 | 使用 fling 手势平滑过渡，不卡顿 |
| 多条跟进记录同一秒 | 按 ID 降序排列 |
| 拨号返回后（onResume） | 自动弹出通话结果反馈面板（详见 07-通话结果反馈面板）；提交走单接口 `POST /api/tenant/calls`（REQ-04：content 非空时原子创建通话+跟进记录），成功后自动刷新跟进时间线与通话记录摘要 |
| 预约日期选择过去时间 | 确认按钮 disabled，日期下方红色 `error-7` 提示 "不能选择过去的时间" |
| 网络断开时提交跟进 | `TDToast` 提示 "网络不可用"，数据保留在 TDPopup 面板中不丢失 |
| 并发编辑同一线索 | 后端返回 409 Conflict → `TDToast` 提示 "线索已被其他人修改，请刷新" |
| 通话记录补正权限 | D 区仅 TM/TA 可见「补正」文字按钮，TE 角色不渲染该按钮（后端 PATCH /calls 为 requireManager） |
| 底部导航条显示条件 | 仅从线索列表页/公海列表页进入时显示（路由携带 listContext）；从日程、通知、看板等入口进入时不显示 |
| 导航条切换时线索已失效 | 目标线索返回 404/403 → 自动跳过，Toast 提示，从 ids 数组中移除，更新计数 |
| 导航条切换时网络异常 | 不切换页面，保持当前线索，Toast 提示重试 |
| 列表数据量大（>200条） | listContext.ids 仅传递当前已加载的 ID 列表；若列表页有分页，传递已加载部分，切换到最后一条时 Toast 提示"已到达当前列表末尾" |
| 切换后返回列表页 | 返回列表页时保持原有滚动位置和筛选状态（列表页状态不因详情页切换而改变） |
| 反馈提交后快捷跳转 | 提交成功提示条 3 秒后自动消失；点击"查看下一个"触发切换；无 listContext 时不显示提示条 |

### 7.2 备注

- TDNavBar 折叠/展开动画使用 `CustomScrollView` + `SliverAppBar` 实现渐变遮罩效果。
- 电话号码的拨号使用 `url_launcher` 调用系统拨号盘 (`tel:` scheme)，避免需要通话权限。
- 跟进时间线的加载更多为**本地切片**：`GET /followups` 一次返回全部记录，前端按批渲染（接口当前不分页）。
- 编辑/删除操作按钮的可见性由**前端计算**：后端不返回 `canEdit`/`canDelete`，前端按「角色 + 记录创建人（userId）+ 创建时间（TE 编辑限 5 分钟窗口）」自行判断渲染。
- 所有轻提示使用 `TDToast`，底部/居中显示，持续 2s。
- 跟进面板中的分类修改为可选操作（传 `categoryId` 同步更新线索分类）；**状态无需手动修改**——提交跟进后后端自动将线索置为 `following`。
- 页面数据缓存策略：从列表页进入时，可将列表页传递的部分数据作为首屏展示，接口返回后刷新差异字段。
- 底部导航条的 `listContext` 由列表页通过路由参数传递，详情页 `initState` 检查是否存在该参数决定是否渲染导航条。
- 导航条切换采用页面内容整体替换（非路由 push），避免页面栈堆积；切换动画使用 `PageView` 或 `AnimatedSwitcher` 实现左右滑动效果。
- 预加载策略：切换到第 N 条时，静默预加载第 N+1 条数据（仅线索详情接口），减少下次切换等待时间。
- 反馈提交后的"查看下一个"提示条与底部导航条共存：提示条位于导航条上方，3 秒后消失。
