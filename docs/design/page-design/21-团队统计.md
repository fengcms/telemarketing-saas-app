## 1. 页面概述

### 1.1 页面定位

团队统计页是 TM/TA 的**核心数据看板**，以可视化图表形式展示团队整体运营状况。包含概览指标、转化漏斗、状态分布、坐席绩效排行、逐日趋势等多个数据模块，帮助管理者快速掌握团队运转全貌。

### 1.2 目标用户

| 角色 | 是否可见 | 说明 |
|------|:--------:|------|
| TE（租户员工） | ❌ | 无权访问，入口不可见 |
| TM（租户经理） | ✅ | 查看团队级统计数据 |
| TA（租户管理员） | ✅ | 查看团队级统计数据 |

### 1.3 核心目标

- 一目了然展示团队线索总量、转化率、公海状况
- 可视化转化漏斗，识别转化瓶颈
- 坐席绩效排行，激励团队竞争
- 逐日趋势折线图，观察数据走势
- 支持日期范围切换，灵活查看不同时段数据

### 1.4 入口路径

```
我的 Tab（Tab 4）→ 个人中心 → 「团队统计」入口（仅 TM/TA 可见）→ 团队统计页（独立 Push 页面）
```

> **入口架构说明（v1.1 整改）**：原"个人中心 → 团队管理区域 → 团队看板卡片"方案已随 [20-团队入口](./20-团队入口.md) 归档废弃。团队模块入口遵循主文档（`APP_PRODUCT_DESIGN.md` §3.1 / §4.8）架构：团队线索池 = 线索 Tab 的 TM/TA 自动 `scope=all` 视图、团队日程 = 日程 Tab 内"我的/团队"切换、**团队统计 = 我的 Tab 个人中心内的独立入口**（本页为该入口对应的唯一独立页面）。

> **变化值（环比）显示规则（v1.1 整改）**：`较昨日` 环比行**仅在日期范围 = "今日"时显示**；选择"本周 / 本月 / 自定义"等其他范围时，该行整体隐藏（不显示"较上一周期"，卡片内只保留标签与数值）。理由：后端 `compareYesterday` 实际口径为"所选范围最后一天 vs 前一天"，在非今日范围下语义含糊，故只在"今日"范围展示。

**四张卡片规格**：

| 卡片 | 标签 | 数值来源 | 变化值来源（仅"今日"范围显示） | 图标 |
|------|------|---------|-----------|------|
| 总线索数 | 总线索 | `total` | `compareYesterday.addedDiff` | `TDIcons.bar_chart` |
| 公海线索 | 公海线索 | `poolTotal` | —（下方显示呆滞数） | `TDIcons.pool` |
| 转化率 | 转化率 | 转化数 / 归属线索数（`convertedCount / Σ agentPerf[].ownedLeads × 100%`） | — | `TDIcons.trending_up` |
| 跟进中 | 跟进中 | `byStatus.following` | `compareYesterday.followupDiff` | `TDIcons.call` |

> **转化率口径（v1.1 整改，产品已确认）**：转化率 = **转化数 / 归属线索数（ownedLeads）**，与后端 `stats` 实现一致（`stats/mine` 的 `myConversionRate = myConverted / myLeadsTotal`，分母为 0 时返回 0）。原"转化数 / 线索池总量（pool）"口径已废弃。注意：后端团队统计接口当前**不直接返回**团队级 `conversionRate` 字段，前端按上述口径用 `convertedCount / Σ agentPerf[].ownedLeads` 计算，分母为 0 时显示 `0.0%`。

**数值样式**：

| 元素 | 字号 | 字重 | 颜色 |
|------|------|------|------|
| 标签 | `fontBodyExtraSmall`（12sp） | Regular（400） | `gray-6`（`#A6A6A6`） |
| 数值 | `fontDisplayLarge`（48sp） | Bold（700） | `gray-12`（`#181818`） |
| 变化值（上升/正向） | `fontBodyExtraSmall`（12sp） | Medium（500） | `success-7`（`#2BA471`）+ ↑ 箭头 |
| 变化值（下降/负向） | `fontBodyExtraSmall`（12sp） | Medium（500） | `error-7`（`#D54941`）+ ↓ 箭头 |
| 变化值（无变化） | `fontBodyExtraSmall`（12sp） | Medium（500） | `gray-6`（`#A6A6A6`）+ "持平" |

**大数字溢出处理（v1.1 补充）**：48sp 大数字在卡片宽度内最多容纳 4 位（含千分位逗号时约 5~6 字符）。数值超过 4 位（即 > 9,999）时按以下规则降级，保证不截断、不换行：

| 数值位数 | 处理 |
|---------|------|
| ≤ 4 位（如 `1234`） | 保持 `fontDisplayLarge`（48sp） |
| 5~7 位（如 `12,345` / `1,234,567`） | 缩小为 `fontDisplayMedium`（36sp），仍带千分位 |
| > 7 位 | 缩写为"万"单位（如 `1234.5万`），字号回落至 48sp |

> 兜底规则：无论哪档字号，数值 `Text` 均设置 `maxLines=1` + `overflow: TextOverflow.ellipsis`（配合 `autoSize` 自适应缩字），确保系统字体放大档位下也不撑破卡片。

## 4. 交互流程

### 4.1 页面初始化

```
用户从「我的 Tab → 个人中心」点击「团队统计」入口

  │   ├─ dateFrom = 本周一（YYYY-MM-DD）
  │   └─ dateTo = 今天（YYYY-MM-DD）
  │
  ├─ 发起请求: GET /api/tenant/stats?dateFrom=...&dateTo=...
  │
  ├─ 请求成功
  │   ├─ 隐藏 TDSkeleton
  │   ├─ 渲染概览卡片（数值从 0 动画滚动到目标值，300ms）
  │   ├─ 渲染转化漏斗（进度条从 0 动画展开到目标宽度，500ms）

  │   └─ 渲染坐席绩效排行（列表项依次淡入，每项间隔 50ms）
  │
  └─ 请求失败
      ├─ 隐藏 TDSkeleton
      └─ 显示错误态（见 5.3）
```

## 6. 接口依赖

> **全局约定（v1.1 整改）**：本章接口格式以 [00-全局API约定](./00-全局API约定.md) 为唯一事实来源。响应统一信封为 `{ "success": true, "data": { ... }, "error": null }`（失败时 `success=false`、`data=null`、`error={code,message}`）；下文"响应字段映射"均指 `data` 内字段。

### 6.1 核心接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 团队统计 | GET | `/api/tenant/stats` | 获取团队级统计数据（仅 TM/TA 可调，TE 调用返回 403 `AUTH_FORBIDDEN`） |

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `dateFrom` | string (YYYY-MM-DD) | ✅ | 起始日期（**必传**，缺失返回 400 `VALIDATION`） |
| `dateTo` | string (YYYY-MM-DD) | ✅ | 结束日期（**必传**，缺失返回 400 `VALIDATION`） |

> 注：所选日期范围内无任何统计数据时，后端返回 `{ success: true, data: { message: "no data in range" } }`（不含统计字段），前端按空态（见 5.2）处理。

**响应字段映射**：

| 响应字段 | 页面使用位置 | 格式化 |
|---------|------------|--------|
| `total` | 概览卡片 - 总线索数 | 千分位（如 `1,234`） |
| `byStatus.pending` | 状态分布饼图 | 整数 + 百分比 |
| `byStatus.assigned` | 状态分布饼图 | 整数 + 百分比 |
| `byStatus.following` | 状态分布饼图 + 概览卡片"跟进中" | 整数 + 百分比 |
| `byStatus.converted` | 状态分布饼图 | 整数 + 百分比 |
| `byStatus.invalid` | 状态分布饼图 | 整数 + 百分比 |
| `byCategory` | MVP 暂不在 APP 展示 | — |
| `byProject` | MVP 暂不在 APP 展示 | — |
| `poolTotal` | 概览卡片 - 公海线索 | 千分位 |
| `staleInPool` | 公海卡片 - 呆滞数 | 整数，>0 时 `warning-7` 高亮 |
| `funnel.pool` | 转化漏斗第一行 | 整数 + 百分比（基准 100%） |
| `funnel.assigned` | 转化漏斗第二行 | 整数 + 百分比 |
| `funnel.following` | 转化漏斗第三行 | 整数 + 百分比 |
| `funnel.converted` | 转化漏斗第四行 | 整数 + 百分比 |
| `agentPerf[]` | 坐席绩效排行列表 | 遍历渲染 |
| `agentPerf[].userId` | 内部标识，不展示 | — |
| `agentPerf[].name` | 排行列表 - 姓名 | 原文 |
| `agentPerf[].ownedLeads` | 排行列表 - 线索数 | 整数 |
| `agentPerf[].followedCount` | 排行列表 - 跟进数 | 整数 |
| `agentPerf[].answeredCount` | 排行列表 - 接通数 | 整数 |
| `agentPerf[].noAnswerCount` | MVP 暂不展示 | — |
| `agentPerf[].convertedCount` | 排行列表 - 转化数 | 整数 |
| `agentPerf[].conversionRate` | 排行列表 - 转化率 | 百分比 1 位小数；口径 = `convertedCount / ownedLeads`（分母为 0 返回 0）。后端未返回该字段时前端按此口径计算 |
| `agentPerf[].avgDuration` | MVP 暂不展示 | — |
| `agentPerf[].lastFollowupAt` | MVP 暂不展示 | — |
| `byAnswerType` | MVP 暂不在 APP 展示 | — |
| `dailyTrend[]` | 逐日趋势折线图 | 遍历渲染 |
| `dailyTrend[].date` | X 轴标签 | `MM-DD` 格式 |
| `dailyTrend[].added` | MVP 暂不展示（可后续加入第四条线） | — |
| `dailyTrend[].followup` | 折线 - 跟进数 | 整数 |
| `dailyTrend[].converted` | 折线 - 转化数 | 整数 |

| `dailyTrend[].total` | MVP 暂不展示 | — |
| `compareYesterday.addedDiff` | 总线索卡片 - 变化值 | 带正负号的整数；**仅日期范围 = "今日"时使用**，其余范围隐藏环比行 |
| `compareYesterday.followupDiff` | 跟进中卡片 - 变化值 | 带正负号的整数；**仅日期范围 = "今日"时使用**，其余范围隐藏环比行 |
| `compareYesterday.convertedDiff` | MVP 暂不展示 | — |

### 6.3 错误码处理

| HTTP 状态码 | 处理方式 |
|------------|---------|
| 200 | 正常渲染 |

## 7. 边界情况与备注

### 7.1 边界情况

| 场景 | 处理方式 |
|------|---------|
| `pool = 0`（无公海线索） | 漏斗所有进度条显示 0%，百分比显示 "—" |
| `total = 0`（无任何线索） | 显示全局空态（见 5.2），概览卡片数值为 0 |
| 日期范围超过 90 天 | 前端限制最大 90 天，`TDCalendarPicker` 超出时提示"最多查看 90 天" |
| `dateFrom > dateTo` | 前端校验拦截，提示"起始日期不能晚于结束日期" |
| 坐席数量为 0 | 坐席绩效区域显示"暂无坐席数据" |
| 坐席数量 1 人 | 正常显示，排行只有第 1 名 |
| 坐席转化率相同 | 按 `convertedCount` 降序二次排序；仍相同则按 `name` 字母序 |
| `dailyTrend` 只有 1 天数据 | 折线图显示单个数据点，不绘制连线，Tooltip 可点击查看 |
| 数值过大（如 total > 99999） | 使用千分位格式化（`123,456`）；若超过 7 位数则缩写为 `12.3万`（另见 §3.3 大数字溢出处理） |
| 转化率为 NaN（converted / ownedLeads，分母为 0） | 显示 `0.0%`（与后端口径一致：分母为 0 返回 0） |
| 日期范围 ≠ 今日（本周/本月/自定义） | 概览卡片隐藏"较昨日"环比行，仅展示标签与数值（见 §3.3 变化值显示规则） |
| 屏幕旋转 | 支持横屏，图表区域自适应宽度；概览卡片变为 4 列 |
| 折叠屏 / 平板 | 概览卡片可改为 4 列 1 行；图表宽度自适应 |
| 系统字体缩放（最大） | 数值文字支持缩放，卡片高度固定，文字超出省略 |

### 7.2 性能考量

| 项目 | 策略 |
|------|------|
| 图表渲染 | 数据量 > 100 个点时启用降采样（每 3 天取一个点） |
| 坐席列表 | 使用 `ListView` 懒加载，避免一次性渲染 |
| 动画 | 图表动画在低端设备上可关闭（检测帧率 < 30fps 时降级为无动画） |
| 缓存 | 同一日期范围的查询结果缓存 5 分钟，避免频繁切换时重复请求 |

### 7.3 图表库选型建议

| 方案 | 说明 |
|------|------|
| 推荐 | [fl_chart](https://pub.dev/packages/fl_chart)（纯 Flutter，可定制 TDesign 风格） |
| 备选 | syncfusion_flutter_charts（功能强大，商业授权） |
| 自定义 | 转化漏斗可用自定义 `TDProgress` 或 `CustomPaint` 自绘，无需第三方库 |
