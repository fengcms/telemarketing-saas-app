# 20 - 团队入口（已归档）

> ⚠️ **本文档已归档（2026-07-22，产品决断）**
>
> 经评审与产品确认，团队功能入口**遵循主设计文档（`APP_PRODUCT_DESIGN.md` §3.1 / §4.8）架构**，不设立独立的"个人中心 → 团队管理"三卡片入口区。原方案（三个入口卡片）与主文档的 Tab 化架构冲突，且造成双入口、功能重叠，故废弃。

## 团队功能的最终入口架构

| 团队功能 | 入口 | 说明 | 详见文档 |
|---------|------|------|---------|
| **团队线索池** | 「线索」Tab（Tab 2） | TM/TA 登录后**自动**展示全部线索（`scope=all&erased=0`），无需独立入口 | [04-线索列表](./04-线索列表.md) / [23-团队线索池](./23-团队线索池.md) |
| **团队日程** | 「日程」Tab（Tab 3） | TM/TA 在日程页顶部显示「团队日程」切换开关，打开后查看团队视图 | [10-日程列表](./10-日程列表.md) / [22-团队日程](./22-团队日程.md) |
| **团队统计** | 「我的」Tab（Tab 4） | TM/TA 在个人中心显示「团队统计」入口（TE 不可见），点击进入独立统计页 | [13-个人中心](./13-个人中心.md) / [21-团队统计](./21-团队统计.md) |

## 归档说明

- 原"团队管理"区域（分隔线 + 区域标题 + 团队看板/团队日程/团队线索池三卡片）整体移除，个人中心不再渲染该区块。
- 角色可见性控制逻辑（TM/TA 才显示团队入口）保留，但仅作用于「我的」Tab 的「团队统计」单一入口，详见 [13-个人中心](./13-个人中心.md)。
- 团队功能整体版本归属为 **v1.1（扩展）**，与主文档附录 A 一致。
- 本文档保留仅作历史参考，不再作为开发依据。
