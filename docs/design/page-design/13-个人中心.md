## 4. 交互流程

### 4.1 页面加载

```
用户点击底部 Tab「我的」
  → 系统切换至个人中心页面
  → 显示 TDSkeleton 加载态（用户信息区 + 统计概览区骨架屏）
  → 并行请求：
      1. 从本地缓存/登录响应中获取用户信息（user 对象）
      2. GET /api/tenant/stats/mine?dateFrom={今天}&dateTo={今天}
         （dateFrom/dateTo 为必传参数，缺失后端返回 VALIDATION；
           概览卡片展示 myLeadsTotal/myFollowed/myAnswered/myConverted）
  → 数据返回后，渲染用户信息区 + 统计概览区
  → 根据用户角色判断是否渲染「团队统计」入口（仅 TM/TA）
  → 隐藏 TDSkeleton，显示完整内容
```

### 4.2 下拉刷新

```
用户在页面顶部下拉
  → 显示 TDRefreshHeader 刷新指示器
  → 重新请求 GET /api/tenant/stats/mine?dateFrom={今天}&dateTo={今天}
  → 数据返回后刷新统计概览区（用户信息区来自本地缓存，无需刷新）
  → 刷新结束，收起 TDRefreshHeader
```

### 4.3 点击功能入口

```
用户点击 TDCell 列表项（如「通话记录」）
  → TDesign 内置点击态反馈
  → 系统执行页面跳转至对应路由
  → 使用系统默认转场动画（右滑入，300ms easeInOut）
```

### 4.4 点击统计概览卡片

```
用户点击「我的业绩」统计卡片
  → TDesign 内置点击态反馈
  → 跳转至个人统计页 /mine/stats（14-个人统计）
```

### 4.5 角色动态显示

```
页面渲染时：
  → 读取当前用户角色（user.role）
  → if role == 'TE': 隐藏「团队统计」入口
  → if role == 'TM' or role == 'TA': 显示「团队统计」入口
```

## 5. 状态设计

### 5.1 加载态

| 区域 | 表现 |
|------|------|
| 用户信息区 | `TDSkeleton`：圆形骨架 + 3 行文字骨架（长/中/短） |
| 统计概览区 | `TDSkeleton`：4 个数字骨架 + 4 个标签骨架 |
| 功能入口 | 正常渲染（不依赖接口，静态配置） |
| 团队统计入口 | 正常渲染（角色信息来自本地缓存） |

### 5.2 空态

| 场景 | 表现 |
|------|------|
| 统计数据全为 0 | 统计概览正常显示 0，不展示空态提示 |
| 用户信息缺失 | `TDAvatar` 显示默认图标，姓名显示"未知用户"，其余字段隐藏 |

### 5.3 错误态

| 场景 | 表现 |
|------|------|
| stats/mine 接口失败 | 统计概览区显示 "数据加载失败，点击重试"（`fontBodyMedium`，`error-7` 色），可点击重新请求 |
| 网络断开 | `TDToast` 提示 "网络连接失败"，带重试操作 |
| Token 过期 | 跳转登录页 |

### 5.4 成功态

页面正常渲染，无额外成功提示。

## 6. 接口依赖

### 6.1 获取个人统计

> 响应信封、错误码等统一遵循 [00-全局API约定](./00-全局API约定.md)。

```
GET /api/tenant/stats/mine?dateFrom=2026-07-22&dateTo=2026-07-22

请求参数：
| 参数     | 类型   | 必填 | 说明                              |
|----------|--------|------|----------------------------------|
| dateFrom | string | 是   | 起始日期 YYYY-MM-DD（概览取当天） |
| dateTo   | string | 是   | 结束日期 YYYY-MM-DD（概览取当天） |

成功响应（{success, data, error} 信封，以下为 data 内字段）：
{
  "success": true,
  "data": {
    "myLeadsTotal": 128,       // 线索总数
    "myFollowed": 356,         // 累计跟进数
    "myAnswered": 210,         // 累计接通数
    "myConverted": 42          // 累计转化数
  },
  "error": null
}
```

### 6.2 用户信息

```
来源：登录响应中的 user 对象（本地缓存）

字段：
{
  "name": "张三",
  "email": "zhangsan@example.com",
  "role": "TE",              // TE / TM / TA
  "tenantName": "XX科技有限公司"
}
```

## 7. 边界情况与备注

| 场景 | 处理方式 |
|------|---------|
| 姓名为空 | `TDAvatar` 显示默认用户图标，姓名区域显示"未设置姓名" |
| 邮箱为空 | 隐藏邮箱行 |
| 所属租户名称过长 | 单行显示，超出部分省略号截断（maxLines=1, overflow=ellipsis） |
| 统计数据为 null | 显示为 "—" |
| 统计数据数值极大 | 超过 9999 显示为 "9999+" 或 "1.2w" 格式 |
| 角色字段异常（非 TE/TM/TA） | 默认按 TE 处理，隐藏团队统计入口 |
| 下拉刷新 | 支持 `TDRefreshHeader` 下拉刷新，重新请求 `stats/mine`（dateFrom/dateTo 取当天）刷新个人统计概览 |
| 页面返回恢复 | 从子页面返回时无需刷新数据（统计为累计值，变化频率低） |
