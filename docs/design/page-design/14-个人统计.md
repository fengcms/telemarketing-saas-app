## 1. 页面概述

| 项目 | 说明 |
|------|------|
| **页面定位** | 展示当前用户的个人业绩统计数据，支持按日期范围筛选查看 |
| **目标用户** | 所有已登录用户（TE / TM / TA） |
| **核心目标** | 让用户清晰了解自己在指定时间段内的业绩表现，包括线索、跟进、接通、转化等核心指标 |
| **页面路由** | `/mine/stats` |
| **入口** | 个人中心 → 统计概览区域点击 |

## 4. 交互流程

### 4.1 页面加载

```
用户从个人中心点击进入
  → 系统跳转至个人统计页面
  → 默认选中「今日」Tab
  → 显示 TDSkeleton 加载态
  → 请求 GET /api/tenant/stats/mine?dateFrom={today}&dateTo={today}
  → 数据返回后：
      - 「今日概况」渲染 myToday 实时数据（固定，不随 Tab 变化）
      - 「数据详情」+ 转化率环形图渲染当前区间数据
  → 隐藏 TDSkeleton
```

### 4.2 切换日期范围

```
用户点击 TDTabBar Tab（如「本周」）
  → Tab 切换动画（indicator 滑动过渡，200ms easeInOut）
  → 「今日概况」区域保持不变（始终为 myToday 实时数据）
  → 「数据详情」+ 转化率环形图区域显示 TDSkeleton 加载态
  → 请求 GET /api/tenant/stats/mine?dateFrom={monday}&dateTo={today}
  → 数据返回后仅刷新「数据详情」+ 转化率环形图区域
  → 隐藏 TDSkeleton
```

### 4.3 自定义日期范围

```
用户点击「自定义」Tab
  → 弹出 TDDialog，内嵌 TDCalendarPicker
  → 用户选择起止日期
  → 点击「确定」
  → 关闭 TDDialog
  → 「数据详情」+ 转化率环形图区域显示 TDSkeleton 加载态（「今日概况」不变）
  → 请求 GET /api/tenant/stats/mine?dateFrom={from}&dateTo={to}
  → 数据返回后仅刷新「数据详情」+ 转化率环形图区域
```

```
用户在 TDCalendarPicker 中点击「取消」
  → 关闭 TDDialog
  → 保持之前的 Tab 选中状态和数据不变
```

## 5. 状态设计

### 5.1 加载态

| 区域 | 表现 |
|------|------|
| 日期选择器 | 正常显示（不依赖接口） |
| 今日概况 | 仅首次加载显示 `TDSkeleton`：2 个卡片骨架（标签骨架 + 数字骨架）；Tab 切换时本区域不重载 |
| 数据详情 | `TDSkeleton`：6 个卡片骨架（标签骨架 + 数字骨架） |
| 转化率环形图 | `TDSkeleton`：圆形骨架 |

### 5.2 空态

| 场景 | 表现 |
|------|------|
| 所有统计值为 0 | 各卡片正常显示 0 / 0%，环形图显示 0% |
| 接口返回空数据 | `TDEmpty` 展示空态插图 + "暂无统计数据"（`fontTitleSmall`，`gray-6`） |

### 5.3 错误态

| 场景 | 表现 |
|------|------|
| 接口请求失败 | `TDEmpty` 展示错误插图 + "数据加载失败"（`fontTitleSmall`，`gray-6`）+ `TDButton(theme: TDButtonTheme.text)` "重试"（`brand-7`） |
| 网络断开 | `TDToast` 提示 "网络连接失败"，带重试操作 |
| 日期范围无效 | `TDToast` 提示 "请选择有效的日期范围" |

### 5.4 成功态

数据正常渲染，无额外成功提示。

## 6. 接口依赖

### 6.1 获取个人统计数据

> 响应信封、错误码等统一遵循 [00-全局API约定](./00-全局API约定.md)。

```
GET /api/tenant/stats/mine?dateFrom=2026-07-22&dateTo=2026-07-22

请求参数：
| 参数      | 类型   | 必填 | 说明                    |
|-----------|--------|------|------------------------|
| dateFrom  | string | 是   | 起始日期 YYYY-MM-DD     |
| dateTo    | string | 是   | 结束日期 YYYY-MM-DD     |

成功响应（{success, data, error} 信封）：
{
  "success": true,
  "data": {
    "myLeadsTotal": 128,          // 当前归属我的线索总量
    "myFollowed": 356,            // 累计跟进数
    "myAnswered": 210,            // 累计接通数
    "myNoAnswer": 146,            // 累计未接数
    "myConverted": 42,            // 累计转化数
    "myConversionRate": 12.5,     // 转化率百分比（后端计算，前端直接展示）
    "myToday": {
      "followupCount": 12,        // 今日新增跟进数（固定用于「今日概况」）
      "answeredCount": 8          // 今日接通电话数（固定用于「今日概况」）
    }
  },
  "error": null
}

注意：
- dateFrom 和 dateTo 为必传参数（缺失后端返回 VALIDATION）
- 后端不返回 byStatus / byCategory / byAnswerType 等分组字段
- 转化率口径（产品已确认）：myConversionRate = myConverted / myLeadsTotal，
  分母为 0 时后端返回 0；页面直接展示该字段，不做二次计算
- myToday 为实时今日数据，与 dateFrom/dateTo 区间无关
```

## 7. 边界情况与备注

| 场景 | 处理方式 |
|------|---------|
| dateFrom > dateTo | 前端校验拦截，`TDToast` 提示"起始日期不能晚于结束日期" |
| 自定义日期范围超过 1 年 | 允许，但请求可能较慢，显示加载指示器 |
| 统计数值为 null | 显示为 "—" |
| 转化率 > 100% | 正常显示（可能存在数据异常），进度条上限 100% |
| 转化率分母为 0 | 转化率显示 "0.0%"，环形图进度为 0 |
| 数值超过 9999 | 显示为 "1.2w" 格式（保留 1 位小数） |
| 今日概况与日期 Tab 关系 | 「今日概况」始终展示 `myToday` 实时数据，**不随所选日期范围变化**（无"当日概况"变形） |
| 快速切换 Tab | 取消前一次请求（dio `CancelToken`），仅保留最新请求结果 |
| 页面返回 | 保留当前 Tab 选择和日期范围状态 |
