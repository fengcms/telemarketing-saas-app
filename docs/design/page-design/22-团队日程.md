## 1. 页面概述

### 1.1 页面定位

团队日程是**「日程」Tab（Tab 3）内 TM/TA 专属的"团队"子视图**（非独立 Push 页面）。TM/TA 在日程页顶部通过「我的 / 团队」切换进入该视图，查看团队所有成员的待办日程。与"我的"视图相比，每条日程额外标注归属人，方便管理者了解团队整体工作安排和负荷情况。

### 1.2 目标用户

| 角色 | 是否可见 | 说明 |
|------|:--------:|------|
| TE（租户员工） | ❌ | 日程 Tab 不显示「我的/团队」切换，固定为"我的"视图 |
| TM（租户经理） | ✅ | 日程 Tab 顶部显示「我的/团队」切换，"团队"视图查看全团队日程 |
| TA（租户管理员） | ✅ | 日程 Tab 顶部显示「我的/团队」切换，"团队"视图查看全团队日程 |

### 1.3 核心目标

- 一览团队所有成员的待办日程
- 按归属人筛选，查看特定员工的工作安排
- 快速识别逾期日程，督促团队跟进
- 通过归属人颜色标签，直观区分不同员工的日程

### 1.4 入口路径

```
日程 Tab（Tab 3）→ 顶部「我的 | 团队」视图切换（仅 TM/TA 可见）→ 切到"团队" → 团队日程视图
```

> **入口架构说明（v1.1 整改）**：原"个人中心 → 团队管理 → 团队日程"独立入口已随 [20-团队入口](./20-团队入口.md) 归档废弃。遵循主文档（`APP_PRODUCT_DESIGN.md` §4.8.2）架构，团队日程并入「日程」Tab 的切换视图：**该视图是日程 Tab 内的子视图而非独立 Push 页面，底部始终保留 `TDBottomTabBar`**；TE 不显示切换，仅"我的"视图。

## 3. 组件规格

### 3.1 顶部栏（Tab 内视图切换头）

> **v1.1 整改**：团队日程为日程 Tab 内子视图，**不使用带返回箭头的独立 `TDNavBar`**（无 Push 转场），顶部栏复用日程 Tab 的页头，核心是「我的 / 团队」视图切换控件。

| 属性 | 值 |
|------|-----|
| 类型 | `TDSegmentedControl`（分段切换）+ 右侧刷新图标 |
| 切换选项 | "我的" / "团队"（**仅 TM/TA 渲染该控件**；TE 不显示切换，页头与个人日程列表一致） |
| 默认选中 | "我的"（TM/TA 与 TE 一致，手动切到"团队"后在会话内保持） |
| 标题字号 | `fontTitleLarge`，24sp |
| 标题字重 | `Medium`（500） |
| 导航图标 | 无返回图标（子视图非独立 Push 页面，返回操作由底部 Tab 切换承担） |
| 右侧操作 | `TDIcons.refresh`，24px，点击刷新当前视图数据 |
| 背景色 | `gray-13`（`#FFFFFF`） |
| 高度 | 48px（不含状态栏） |
| 底部 TabBar | 始终保留 `TDBottomTabBar`（首页/线索/日程/我的） |

**归属人颜色标签**：

```
┌──────┐
│  ●   │  ← 10px 圆形色块，位于卡片左侧
└──────┘
```

| 属性 | 值 |
|------|-----|
| 形状 | 圆形，10px |
| 位置 | 卡片左侧，垂直居中于第一行 |
| 颜色 | 由归属人 ID 哈希生成，确保同一人颜色一致 |

**颜色生成规则**：

| 预设颜色池 | 色值 |
|-----------|------|
| 红 | `#E53935` |
| 蓝 | `#1E88E5` |
| 绿 | `#43A047` |
| 橙 | `#FB8C00` |
| 紫 | `#8E24AA` |
| 青 | `#00ACC1` |
| 粉 | `#D81B60` |
| 棕 | `#6D4C41` |

> 算法：`colorPool[hashCode(userId) % colorPool.size]`，确保同一 userId 始终映射到同一颜色。

**卡片内部元素**：

| 子元素 | 规格 |
|--------|------|
| 归属人姓名 | `fontBodyExtraSmall`，12sp，`medium`，颜色同归属人圆点色，与圆点同行 |
| 日程标题 | `fontBodyLarge`，16sp，`medium`，`gray-12`，单行省略 |
| 关联线索电话 | `fontBodySmall`，12sp，`gray-6`，格式 "📞 13812341234"；**电话完整展示，不脱敏** |
| 时间 | `fontBodySmall`，12sp，`gray-6`，格式 "🕐 HH:mm" |
| 状态标签 | `TDTag`，`fontCaption`（10sp），胶囊形背景标签（见下表） |
| 逾期标记 | 当 `status=pending` 且 `scheduledAt < now` 时显示 `TDIcons.time`（`error-7` 色）+ "已逾期" `error-7` 文字 |

> **不脱敏说明（v1.1 整改）**：原"对他人日程关联电话脱敏"的设计已删除。理由：TM/TA 通过 `scope=all` 权限本身就能查看全团队线索的完整数据（主文档 §4.3.1 明确日程列表 phone 不脱敏），对下属日程关联电话脱敏既无安全增益、又妨碍主管直接回拨跟进，与权限模型自相矛盾。故团队视图中日程内容（标题、备注、关联电话）均按后端返回值完整展示。

**状态标签颜色**：

| 状态 | 文案 | 背景色 | 文字色 |
|------|------|--------|--------|
| pending（未逾期） | 待办 | `brand-1`（`#F2F3FF`） | `brand-7`（`#0052D9`） |
| pending（已逾期） | 已逾期 | `error-7` 10%（`#FDECEA`） | `error-7`（`#D54941`） |
| completed | 已完成 | `success-7` 10% | `success-7`（`#2BA471`） |
| cancelled | 已取消 | `gray-1` | `gray-6` |

## 4. 交互流程

### 4.1 页面初始化

```
用户在日程 Tab（Tab 3）顶部切换至"团队"视图（仅 TM/TA 可见该切换）
  │
  ├─ 显示 TDSkeleton 骨架屏
  │
  ├─ 并行发起请求:
  │   ├─ GET /api/tenant/schedules（TM/TA 调用自动返回全团队日程）
  │   └─ GET /api/tenant/options/users（获取成员列表，用于筛选下拉）
  │       └─ 若成员列表已缓存则跳过此请求
  │       └─ 注意：不可用 /api/tenant/users（TA 专属接口，TM 调用返回 403）
  │
  ├─ 请求成功
  │   ├─ 隐藏 TDSkeleton
  │   ├─ 计算统计摘要（今日待办数、逾期数）
  │   ├─ 渲染统计摘要条
  │   ├─ 按 scheduledAt 日期分组
  │   ├─ 为每个成员分配颜色
  │   ├─ 渲染日程卡片列表
  │   └─ 默认显示"今天"的日程
  │
  └─ 请求失败
      └─ 显示错误态
```

### 4.2 员工筛选

```
用户点击员工筛选下拉框
  │
  ├─ 展开 TDPicker 下拉菜单，显示所有员工 + "全部成员"选项
  │
  ├─ 用户选择某个员工（如"张三"）
  │   │
  │   ├─ 下拉框关闭，显示选中的员工姓名 + 颜色圆点
  │   │
  │   ├─ 列表使用动画过渡
  │   │
  │   ├─ 前端过滤日程列表（仅显示该员工的日程）
  │   │   └─ 无需重新请求后端（数据已全部加载）
  │   │
  │   ├─ 更新统计摘要条（仅统计该员工的数据）
  │   │
  │   └─ 若该员工无日程 → 显示筛选后空态
  │
  └─ 用户选择"全部成员"
      ├─ 恢复显示全部日程
      └─ 统计摘要条恢复为全团队数据
```

### 4.3 日期筛选

```
用户点击日期筛选（如"明天"）
  │
  ├─ 选中态切换（TDTagGroup 动画）
  │
  ├─ 列表滚动到对应日期分组
  │   ├─ "今天" → 滚动到今天分组
  │   ├─ "明天" → 滚动到明天分组
  │   ├─ "本周" → 滚动到顶部，显示本周所有日期
  │   └─ "自定义" → 弹出 TDCalendarPicker → 选择后滚动到对应日期
  │
  └─ 注意：日期筛选不改变数据范围，仅滚动定位
      └─ 所有日程数据一次性加载（按后端分页，前端按日期分组展示）
```

### 4.4 点击日程卡片

```
用户点击某条日程卡片
  │
  ├─ 触发 TDesign 内置水波纹反馈
  │
  ├─ 导航至日程详情页（Push）
  │   ├─ 传递 scheduleId
  │   └─ 日程详情页展示完整信息（标题、内容、关联线索、操作按钮等）
  │
  └─ TM/TA 对他人日程的操作权限（v1.1 整改，对齐后端 schedules.ts）:
      ├─ ✅ 可查看日程完整内容（不脱敏）及关联线索信息
      ├─ ✅ 可标记完成：POST /api/tenant/schedules/:id/complete（pending → completed）
      ├─ ✅ 可取消：POST /api/tenant/schedules/:id/cancel（pending → cancelled）
      ├─ ✅ 可重新打开：POST /api/tenant/schedules/:id/reopen（completed/cancelled → pending）
      ├─ ❌ 不可编辑他人日程的 title / content / scheduledAt（APP 对他人日程仅开放状态操作）
      └─ ❌ 不可删除他人日程（后端仅允许 TA 或日程归属人删除；TM 删除他人日程返回 403）
```

> **权限说明**：后端对 TM/TA 开放团队日程的状态流转（complete/cancel/reopen 仅限制 TE 操作他人日程），故详情页对他人日程展示"完成 / 取消 / 重新打开"按钮；`PATCH /api/tenant/schedules/:id` 仅支持 `scheduledAt/title/content` 字段（不支持 status 变更），且 APP 端不对他人日程暴露编辑入口——团队视图对他人日程**仅状态操作**。

## 6. 接口依赖

> **全局约定（v1.1 整改）**：本章接口格式以 [00-全局API约定](./00-全局API约定.md) 为唯一事实来源。响应统一信封为 `{ "success": true, "data": { ... }, "error": null }`；列表接口 `data` 为分页体 `{ items, total, page, size, pages }`（请求参数 `page`/`size`）。下文"响应字段映射"均指 `data.items[]` 内字段。

### 6.1 核心接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 团队日程列表 | GET | `/api/tenant/schedules` | TM/TA 调用返回全团队日程（TE 调用仅返回本人日程） |
| 成员列表（筛选下拉） | GET | `/api/tenant/options/users` | 轻量下拉接口（id+name+role），全角色可读。**不可用 `/api/tenant/users`**（挂 `requireTA`，TM 调用返回 403 `AUTH_FORBIDDEN`） |
| 标记完成 | POST | `/api/tenant/schedules/:id/complete` | pending → completed；TM/TA 可操作团队日程，TE 仅限本人 |
| 取消 | POST | `/api/tenant/schedules/:id/cancel` | pending → cancelled；权限同上 |
| 重新打开 | POST | `/api/tenant/schedules/:id/reopen` | completed/cancelled → pending；权限同上 |

> **注意 1**：`GET /api/tenant/schedules` 在 TM/TA 调用时自动返回全团队数据，无需额外传 `scope` 参数，后端根据用户角色自动判断数据范围。
> **注意 2**：状态变更**不通过** `PATCH /:id`（PATCH 仅支持 `scheduledAt`/`title`/`content` 字段编辑），而是走上述 complete/cancel/reopen 状态操作端点。

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `page` | int | 否 | 页码，默认 1 |
| `size` | int | 否 | 每页条数，默认 20（最大 200） |
| `status` | string | 否 | 状态筛选（`pending`/`completed`/`cancelled`）。**注意：不传 `status` 或 `status__in` 时后端默认只返回 `pending`** |
| `status__in` | string | 否 | 多状态查询（如 `status__in=pending,completed,cancelled`）；团队视图需展示全部状态时使用 |
| `userId` | string | 否 | 按归属人筛选（后端支持的查询字段；本页当前为前端过滤，见 §4.2） |
| `q` | string | 否 | 搜索关键词（跨 `title` + `content` + 关联线索 `name`） |

**分页响应体**（`data` 内）：

```json
{ "items": [ /* 日程对象，见下 */ ], "total": 128, "page": 1, "size": 20, "pages": 7 }
```

**响应字段映射**（日程对象）：

| 响应字段 | 页面使用位置 | 格式化 |
|---------|------------|--------|
| `id` | 点击跳转详情时传递 | — |
| `title` | 日程卡片 - 标题 | 原文，单行省略 |
| `content` | 日程详情页 | — |
| `scheduledAt` | 日程卡片 - 时间 + 日期分组依据 | **int Unix 秒**（非 ISO8601）；日期部分用于分组，时间部分显示 "HH:mm" |
| `status` | 日程卡片 - `TDTag` 状态标签 | 映射为中文标签 + 对应颜色 |
| `lead` | 日程卡片 - 关联线索电话 | `lead.phone` **完整展示（不脱敏）**；线索被擦除时 `lead` 为 `null` |
| `lead.name` | 日程详情页 | — |
| `userId` | 归属人颜色映射 key + 归属人姓名 | 列表**仅返回 `userId`**（无 `user` 对象）；归属人姓名用 `GET /api/tenant/options/users` 缓存按 `userId` 映射 |
| `createdAt` | 日程详情页 | int Unix 秒 |

### 6.2 成员列表接口（筛选下拉）

接口：`GET /api/tenant/options/users`（全角色可读；**不要用 `/api/tenant/users`**——该接口挂 `requireTA`，TM 调用返回 403）。

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| — | — | — | 无参数，返回当前租户下全部成员（按创建时间倒序，**上限 100 条**，无分页） |

**响应体**（`data` 为**直接数组**，不是 `users[]` 包裹，也不是分页体）：

```json
{ "success": true, "data": [ { "id": "uuid", "name": "张三", "role": "TE" }, ... ], "error": null }
```

| 响应字段 | 页面使用位置 | 说明 |
|---------|------------|------|
| `data[].id` | 筛选下拉选项 value + 归属人映射 key | 成员 ID |
| `data[].name` | 筛选下拉选项 label + 卡片归属人姓名 | 成员姓名 |
| `data[].role` | 筛选下拉选项辅助信息 | 角色标签（TE/TM/TA） |

> 成员列表数据首次加载后缓存至内存，后续切换筛选时不重新请求。下拉刷新时也不刷新成员列表（成员变动需重新进入页面或重新登录）。

### 6.3 请求时机

| 时机 | 请求 |
|------|------|
| 首次切换到"团队"视图 | `GET /api/tenant/schedules` + `GET /api/tenant/options/users`（若未缓存） |
| 切换员工筛选 | 前端过滤，不发请求 |
| 切换日期筛选 | 前端滚动定位，不发请求 |
| 下拉刷新 | `GET /api/tenant/schedules`（当前筛选条件） |
| 无限滚动 | `GET /api/tenant/schedules?page=N` |

### 6.4 错误码处理

| HTTP 状态码 | 处理方式 |
|------------|---------|
| 200 | 正常渲染 |
| 401 / 403 | 跳转登录页 |
| 500 | 显示错误态 + 重试按钮 |
| 网络超时 | 显示错误态 + 重试按钮 |

## 7. 边界情况与备注

### 7.1 边界情况

| 场景 | 处理方式 |
|------|---------|
| 日程关联的线索已被擦除 | `lead` 字段为 `null`，卡片中不显示电话行，显示"线索已删除"灰色文字 |
| 日程无关联线索 | `lead` 为 `null`，卡片中不显示电话行，仅显示标题和时间 |
| 员工数量为 0 | 筛选下拉仅显示"全部成员"，不可选 |
| 员工数量 > 20 | 下拉菜单内部可滚动 |
| 日程跨多天 | 按 `scheduledAt` 的日期部分分组，每条日程只出现在对应日期下 |
| 同一天日程数量很多（> 50） | 正常渲染，使用 `ListView` 懒加载 |
| 日期筛选选择过去的日期 | 正常显示该日期的日程（含已完成/已取消），逾期标记仅对 `pending` 状态生效 |
|  scheduledAt 为 null | 不应出现（后端必填），若出现则归入"未安排时间"分组，显示在列表末尾 |
| 系统时区问题 | `scheduledAt` 为 **int Unix 秒**（时区无关），前端按设备本地时区转换显示 |
| 视图切换（"我的" ↔ "团队"） | 切换为 Tab 内子视图切换（非页面 Push）：切到"团队"隐藏返回按钮、加载全团队数据；切回"我的"恢复个人日程；底部 `TDBottomTabBar` 始终保留 |
| 搜索功能 | 搜索框位于筛选栏下方（可选实现），支持搜索 `title` + `content` + `lead.name` |

### 7.2 与个人日程的差异对照

| 特性 | 个人日程（"我的"视图） | 团队日程（"团队"视图，TM/TA） |
|------|-------------|----------------|
| 数据范围 | 仅自己的日程 | 全团队日程 |
| 归属人显示 | 不显示 | 显示归属人姓名 + 颜色标签 |
| 成员筛选 | 无 | 可按成员筛选 |
| 操作权限 | 可完成/取消/编辑/删除 | 对他人日程：可查看（不脱敏）+ 可完成/取消/重新打开（状态操作）；**不可编辑他人日程的 title/content/time**；删除仅 TA 或日程归属人可操作 |
| 新建日程 | 可新建 | 不可在列表页新建（需从线索详情进入；TM/TA 可替任意线索建并指定归属员工） |
| 统计摘要 | 无 | 显示今日待办 + 逾期统计 |

### 7.3 备注

| 项目 | 说明 |
|------|------|
| 颜色分配一致性 | 同一员工在不同页面（团队日程、团队线索池）中的颜色应保持一致，颜色映射逻辑应提取为公共工具方法 |
| 日程详情页复用 | 点击日程卡片进入的详情页复用现有日程详情页组件；对他人日程仅显示"完成 / 取消 / 重新打开"状态按钮，隐藏"编辑 / 删除"（编辑入口仅本人日程可见，删除仅 TA 或归属人可见，见 §4.4） |
| 与日程 Tab 联动 | 从日程 Tab 切换到团队模式时，保持当前的日期筛选状态；切换回个人模式时恢复 |
| 性能 | 团队日程数据量通常可控（10~50 人 × 每人日均 3~5 条 = 30~250 条），一次性加载无性能问题 |
| 与产品设计文档对应 | 对应 `APP_PRODUCT_DESIGN.md` §3.1 页面树中"团队日程视图"节点，及 §4.3 待办日程模块 |
