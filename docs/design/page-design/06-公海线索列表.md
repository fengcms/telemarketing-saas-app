## 1. 页面概述

### 1.1 页面定位

公海线索列表页为 TE（电销坐席）提供从公海中自主领取线索的能力。公海线索是指尚未分配给任何坐席的待跟进线索（`status=pending` 且 `owner_id IS NULL`）。TE 可浏览公海线索详情后，决定是否领取该线索纳入自己的跟进池。

### 1.2 目标用户

| 角色 | 可见性 | 说明 |
|------|--------|------|
| TE（电销坐席） | 可见 | 浏览公海线索、领取线索 |
| TM（电销主管） | 不可见 | TM 通过管理后台分配线索，不使用公海领取功能 |
| TA（电销管理员） | 不可见 | 同 TM |

### 1.3 核心目标

- 让 TE 自主从公海中选取适合的线索进行跟进
- 提升线索利用率，减少资源浪费
- 领取后线索自动进入"我的线索"，TE 可立即开展跟进

### 1.4 显示条件

- 仅当租户配置 `allowSelfClaim=true` 时，顶部切换 Tab 才显示
- 若 `allowSelfClaim=false`，则不显示切换 Tab，页面退化为标准线索列表（参见 04-线索列表）

### 1.5 数据范围

- `scope=public`
- 仅展示 `status=pending AND owner_id IS NULL` 的线索
- 自动排除 `erased=1`（已擦除）和 `is_blocked=1`（禁拨名单）的记录

### 2.3 领取方式说明（一键领取，无确认弹窗）

> **产品决策**：领取为双入口——① 公海卡片上的「领取该线索」按钮（一键领取）；② 公海线索详情页底部的「领取」按钮。
> 两处点击后**直接发起领取请求，不弹出确认弹窗**（领取为低风险、可即时感知结果的操作，二次确认会增加操作成本）。领取结果统一以 `TDToast` + 卡片移除动画反馈（见 §2.4、§4.5）。

## 4. 交互流程

### 4.1 切换到公海线索

```
用户操作                          系统响应
──────────────────────────────────────────────────────────
1. 在「我的线索」模式          → 2. 点击顶部 TDTabBar「公海线索」Tab
                               → 3. TDTabBar 指示器滑动到「公海线索」
                               → 4. 搜索框占位文字切换为 "搜索公海线索..."
                               → 5. 列表区域显示 TDSkeleton
                               → 6. 发起请求:
                                  GET /api/tenant/leads?scope=public&status=pending&erased=0&page=1&size=20
                               → 7. 接口返回 → 隐藏 TDSkeleton → 渲染公海线索卡片
                               → 8. 若总数 = 0 → 显示 TDEmpty 公海空态
```

### 4.2 切换回我的线索

```
用户操作                          系统响应
──────────────────────────────────────────────────────────
1. 在「公海线索」模式          → 2. 点击顶部 TDTabBar「我的线索」Tab
                               → 3. TDTabBar 指示器滑动到「我的线索」
                               → 4. 恢复「我的线索」列表
                                  (保留之前的搜索/筛选/排序状态和滚动位置)
```

### 4.3 公海线索搜索

```
用户操作                          系统响应
──────────────────────────────────────────────────────────
1. 在公海模式下输入搜索关键词  → 2. 行为同线索列表搜索（TDSearchBar 防抖 500ms + 回车）
                               → 3. 请求参数叠加 q 参数:
                                  scope=public&status=pending&erased=0&q={keyword}
                               → 4. 搜索结果替换当前列表
                               → 5. 清除搜索 → 恢复公海原始列表
```

### 4.4 公海线索筛选

```
用户操作                          系统响应
──────────────────────────────────────────────────────────
1. 在公海模式下点击漏斗图标    → 2. 弹出筛选面板 TDPopup
                               → 3. 状态筛选默认锁定为 "待跟进" (不可更改)
                                  其他筛选项（分类、项目、时间段）正常可选
                               → 4. 确定后叠加筛选条件请求
```

### 4.5 领取线索流程（一键领取）

```
用户操作                          系统响应
──────────────────────────────────────────────────────────
1. 点击卡片上的「领取该线索」 → 2. 按钮进入 loading 态（防重复点击，不弹确认弹窗）
                               → 3. POST /api/tenant/leads/:id/claim
                               → 4. 成功 (响应 data: {id, ownerId, status:"assigned"}):
                                  → TDToast 成功提示 "领取成功，已加入您的线索池" (2s)
                                  → 该卡片从公海列表中移除（淡出动画，300ms）
                                  → 自动切换到 TDTabBar「我的线索」Tab
                                  → 「我的线索」列表刷新（从第一页重新加载）
                               → 5. 失败（业务错误：线索已被他人领取 / 已不在公海，
                                  后端返回 400 VALIDATION）:
                                  → 按钮恢复
                                  → TDToast 提示 "该线索已被领取"
                                  → 自动刷新公海列表（该卡片随之消失）
                               → 6. 失败（禁拨名单，422 BLOCKLIST_REJECTED）:
                                  → TDToast 提示 "该线索在禁拨名单中，无法领取"
                               → 7. 失败（网络错误 / 500）:
                                  → 按钮恢复
                                  → TDToast 提示 "网络异常，请重试" / "服务异常，请稍后重试"
```

> 从公海线索**详情页**底部「领取」按钮发起领取时，调用同一接口、同一成功/失败处理；成功后返回公海列表并刷新（该卡片已不在公海）。

### 4.6 卡片点击（进入详情）

```
用户操作                          系统响应
──────────────────────────────────────────────────────────
1. 点击卡片信息区域（非按钮） → 2. 跳转至线索详情页
                               → 路由: /lead/:id?scope=public
                               → 详情页以公海模式展示:
                                  - 拨号按钮替换为「领取」按钮（领取第二入口，
                                    与卡片一键领取同接口、同成功/失败处理，见 §4.5）
                                  - 不显示归属人
                                  - 不显示跟进时间线（公海线索无跟进记录）
```

### 4.7 无限滚动加载

```
用户操作                          系统响应
──────────────────────────────────────────────────────────
1. 列表滚动到底部              → 2. TDLoadMore 自动触发加载下一页
                               → 3. 底部 TDLoadMore 显示
                               → 4. 接口返回 → 追加新卡片
                               → 5. 无更多数据 → 显示"已加载全部公海线索"
```

### 4.8 下拉刷新

```
用户操作                          系统响应
──────────────────────────────────────────────────────────
1. 公海列表顶部下拉            → 2. TDRefreshHeader 动画
                               → 3. 重置 page=1，重新请求
                               → 4. 接口返回 → 替换全部列表
                               → 5. 隐藏 TDRefreshHeader
```

| 文字与插图间距 | `spacer16` (16px) |
| 整体垂直居中 | 在列表区域居中 |

### 5.4 错误态

### 5.5 领取中状态

| 属性 | 规格 |
|------|------|

### 6.1 获取公海线索列表

```
GET /api/tenant/leads
```

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `scope` | string | 自动 | 固定 `public` |
| `status` | string | 自动 | 固定 `pending`（裸字段 = 等值过滤） |
| `page` | int | 否 | 页码，默认 1 |
| `size` | int | 否 | 每页条数，默认 20 |
| `erased` | int | 自动 | 固定 `0` |
| `q` | string | 否 | 搜索关键词 |
| `categoryId` | string | 否 | 分类筛选（裸字段 = 等值过滤） |
| `projectId` | string | 否 | 项目筛选（裸字段 = 等值过滤） |
| `updatedAt__gte` | int | 否 | 时间段筛选-起始（Unix 秒） |
| `updatedAt__lte` | int | 否 | 时间段筛选-截止（Unix 秒） |
| `sort` | string | 否 | 排序字段，默认 `-updatedAt`（`-` 前缀 = 降序） |

**响应结构（成功 200，全局分页信封）：**

```json
{

  "data": {
    "items": [
      {
        "id": 10055,
        "name": "王*华",
        "phone": "159****5678",
        "company": "YY贸易公司",
        "status": "pending",
        "category": "商铺",
        "project": {
          "id": 8,
          "name": "翠湖天地"
        },
        "owner": null,
        "isBlocked": false,
        "lastFollowupAt": null,
        "nextFollowupAt": null,
        "updatedAt": 1752566400
      }
    ],
    "total": 42,
    "page": 1,
    "size": 20,
    "pages": 3
  },
  "error": null

```

> 时间字段为 Unix 秒级 int 时间戳，展示层格式化为相对时间（"3天前"）或 `YYYY-MM-DD HH:mm`。

### 6.2 领取线索

```
POST /api/tenant/leads/:id/claim
```

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | int | 路径参数 | 线索 ID |

**请求体：** 无（或空 body）

**响应结构（成功 200，全局信封）：**

```json
{
  "success": true,
  "data": {
    "id": 10055,
    "ownerId": "7",
    "status": "assigned"
  },
  "error": null
}
```

> 后端**不返回** `claimedAt` 字段（领取时间即 `updatedAt`/`assignedAt`，由服务端记录）；领取成功后线索状态为 `assigned`（已分配），不再是 `pending`。

**响应结构（失败 400：线索已被领取 / 已不在公海，错误码 `VALIDATION`）：**

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "VALIDATION",
    "message": "only pool leads can be claimed"
  }
}
```

**响应结构（失败 422：禁拨名单，错误码 `BLOCKLIST_REJECTED`）：**

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "BLOCKLIST_REJECTED",
    "message": "该线索在禁拨名单中，无法领取"
  }
}
```

> 其他失败：非 TE 角色或租户未开启 `allowSelfClaim` → 403 `AUTH_FORBIDDEN`；线索不存在 → 404 `NOT_FOUND`。

## 7. 边界情况与备注

### 7.1 边界情况

| 场景 | 处理方式 |
|------|----------|
| `allowSelfClaim=false` | 不显示切换 TDTabBar，页面退化为标准线索列表 |
| TE 角色以外的用户访问 | TM/TA 不显示切换 TDTabBar，仅 TE 可见 |
| 公海线索被他人抢先领取 | 领取接口返回 400 `VALIDATION` → `TDToast` 提示 "该线索已被领取" + 刷新列表 |
| 公海线索在禁拨名单中 | 领取按钮 disabled + 卡片右上角显示禁拨标记（`TDIcons.lock` 小锁图标） |
| 公海线索被管理员擦除 | 列表中不显示（后端已排除 `erased=1`） |
| 快速连续点击领取 | `TDButton` loading 态防重复点击 |
| 领取成功后网络断开 | `TDToast` 已显示"领取成功"，切换 Tab 后若刷新失败显示 `TDEmpty` 错误态 |
| 公海线索总数为 0 | 显示 `TDEmpty` 公海空态 |
| 从公海详情页返回列表 | 若该线索已被领取，列表中该卡片已移除 |
| 切换 Tab 时正在加载 | 取消当前请求，切换后发起新请求 |
| 搜索+筛选叠加后无结果 | 显示 `TDEmpty` 搜索空态 |
| 公海线索卡片中项目名为空 | 第三行仅显示分类 `TDTag`，不显示项目名 |
| 公海线索卡片中分类为空 | 第三行仅显示项目名，不显示分类 `TDTag` |
| 分类和项目均为空 | 第三行不显示 |
| 领取后「我的线索」列表数量变化 | 切换 Tab 后从第一页重新加载，列表数量 +1 |
| 公海线索被管理员重新分配给他人 | 该线索不再出现在公海列表中（`owner_id IS NOT NULL`） |

### 7.2 与线索列表页的组件复用

| 组件 | 复用情况 | 差异说明 |
|------|----------|----------|
| TDNavBar + TDSearchBar | 完全复用 | 占位文字不同 |
| 排序按钮 + TDPopup | 完全复用 | — |
| 筛选面板 TDPopup | 复用 | 状态选项默认锁定 "待跟进" |
| 筛选标签栏 TDCheckTag 组 | 完全复用 | — |
| 线索卡片（信息区域） | 部分复用 | 去掉归属人行和 nextFollowupAt 徽章 |
| 底部 TDLoadMore | 完全复用 | — |
| TDSkeleton 骨架屏 | 部分复用 | 增加领取按钮骨架 |
| TDEmpty 空态 / 错误态 | 部分复用 | 文案不同 |

### 7.3 备注

- 公海线索列表与我的线索列表共享同一个页面（Flutter Widget），通过 `TDTabBar` 切换数据源（`scope` 参数），而非两个独立页面。这样可最大化组件复用。
- 领取操作成功后自动切换到「我的线索」Tab 并刷新，这是核心交互，确保用户能立即看到领取结果。
- 公海线索的搜索/筛选/排序状态与「我的线索」独立保存，切换 Tab 时互不影响。
- 公海线索卡片点击进入详情页时，需传递 `scope=public` 参数，详情页据此调整展示逻辑（隐藏跟进时间线、拨号按钮替换为领取按钮等）。
- 禁拨名单的校验由后端在领取接口中执行，前端仅做展示辅助（如已知的禁拨线索显示锁定标记）。
- 公海线索数量角标（TDTabBar 上的 `TDBadge` 数字）随公海列表请求（含下拉刷新）返回的 `total` 更新。本项目**不提供 WebSocket/SSE/推送**，领取成功后将卡片从本地列表移除即可保证即时反馈，数量准确性由下次刷新兜底。
- 所有颜色值均引用 TDesign Flutter 设计令牌（Design Tokens），实际开发中通过 `TDTheme.of(context)` 获取，不硬编码 HEX 值。
