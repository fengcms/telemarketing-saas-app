## 1. 页面概述

| 项目 | 说明 |
|------|------|
| **页面定位** | 新建日程表单页，用于为线索创建跟进日程/待办提醒 |
| **目标用户** | TE（电销员）、TM（电销主管）、TA（电销管理员） |
| **核心目标** | 1. 快速为线索创建下次跟进日程；2. 支持从线索详情页或跟进反馈面板两种入口进入，部分字段预填；3. TM/TA 可替下属员工创建日程 |
| **入口** | **入口 1**：线索详情页 → "预约下次跟进"按钮 → 进入新建日程页（`leadId` 预填）；**入口 2**：跟进反馈面板 → 快捷日程按钮（见 `07-通话反馈面板.md`）→ 进入新建日程页（`leadId` + 面板已输入的跟进内容预填到「内容」字段；**不预填 `scheduledAt`**，计划时间由用户在本页主动选择）。两个入口均为**全屏 Push 页面**（产品决策，不采用 BottomSheet 形式） |
| **权限说明** | TE：只能给自己归属的线索建日程；TM/TA：可替任意线索建，可选指定归属员工 |

#### 3.3.1 快捷时间 chip

> 位于计划时间输入框下方，帮助用户（尤其是入口 2 场景）一键选定常见跟进时间，免去打开两级选择器的操作。

| 属性 | 规格 |
|------|------|
| 组件 | `TDCheckTag`（可选中标签），水平排列，间距 `spacer8`（8px），外边距上 `spacer8`（8px） |
| 预设项 | 「明天 10:00」「明天 15:00」「后天 10:00」三项（按北京时间计算目标日的对应时刻） |
| 未选中样式 | `gray-1`（#F3F3F3）背景，`gray-12`（#181818）文字，`fontBodyMedium`（14sp），圆角 `radiusDefault`（6px） |
| 选中样式 | `brand-1`（#F2F3FF）背景，`brand-7`（#0052D9）文字 + 描边 |
| 点击行为 | 点击未选中 chip → 选中该 chip 并将计划时间设为对应时刻（输入框同步显示）；再次点击已选 chip → 取消选中，计划时间清空（回到"请选择计划时间"） |
| 互斥关系 | chip 之间互斥（单选）；用户通过输入框打开 `TDCalendarPicker`/`TDDatePicker` 并确认自定义时间后 → 清除所有 chip 选中态（自定义时间不属于预设项） |
| 过去时间 | 预设项均为未来时间（明天/后天），无需逾期提示；若用户跨天停留页面导致 chip 时间已过去，提交时按 §3.3"选择过去时间"规则提示 |

## 4. 交互流程

### 4.1 页面初始化（入口 1：线索详情页）

```
用户操作                          系统响应
─────────                        ─────────
1. 在线索详情页点击         →    2. 页面从右侧滑入（300ms，`Curves.easeInOut`）
   "预约下次跟进"            →    3. 接收参数：leadId
                            →    4. 请求线索摘要 GET /api/tenant/leads/:leadId（获取姓名、电话、项目）
                            →    5. 渲染关联线索信息（只读）
                            →    6. 计划时间默认填充为 当前时间 + 1 小时（取整到 5 分钟）
                            →    7. 若用户角色为 TM/TA → 显示归属人选择器，请求
                                   GET /api/tenant/options/users 加载用户列表
                            →    8. 若用户角色为 TE → 隐藏归属人选择器
```

### 4.2 页面初始化（入口 2：跟进反馈面板）

```
用户操作                          系统响应
─────────                        ─────────
1. 在跟进面板点击            →    2. 页面从右侧滑入（全屏 Push，与入口 1 一致；
   快捷日程按钮                    产品决策：不采用 BottomSheet）
                            →    3. 接收参数：leadId + content（面板已输入的
                                   跟进内容）；不接收 scheduledAt
                            →    4. 请求线索摘要（同入口 1）
                            →    5. 「内容」字段预填为传入的 content（可修改）
                            →    6. 计划时间为空（placeholder"请选择计划时间"），
                                   需用户主动选择（快捷 chip 或选择器）
                            →    7. 其他逻辑同入口 1
```

注：用户也可通过快捷时间 chip 一键选定时间（§3.3.1），无需打开选择器；
    通过选择器确认自定义时间后，清除所有 chip 选中态。

### 4.5 提交创建

```
用户操作                          系统响应
─────────                        ─────────
1. 点击「创建日程」按钮     →    2. 表单校验：
                                   a. 计划时间是否已选 → 未选：输入框标红 + 提示"请选择计划时间"
                                   b. 标题长度 ≤ 200 → 超长：阻止提交
                                   c. 内容长度 ≤ 2000 → 超长：阻止提交
                            →    3. 校验通过：
                                   a. 按钮进入 loading 状态（文字消失，显示 `TDLoading`）
                                   b. 按钮禁用，防止重复提交
                            →    4. 请求 POST /api/tenant/schedules
                                   Body: {
                                     "leadId": "lead_001",
                                     "scheduledAt": 1784098800,  // int Unix 秒
                                     "title": "跟进通话结果",
                                     "content": "...",
                                     "userId": "user_001"  // TM/TA 传，TE 不传
                                   }
                                   （scheduledAt 由所选本地时间换算：
                                     DateTime → Unix 秒（UTC），见 00 文档 §7；
                                     示例 1784098800 = 2026-07-15 15:00 北京时间）
                            →    5. 成功（HTTP 201，响应 { "success": true,
                                   "data": { "id": "sch_002", "leadId": "lead_001" },
                                   "error": null }）：
                                   a. `TDToast` "日程已创建"
                                   b. 页面关闭（带动画返回上一页）
                                   c. 上一页（线索详情页/跟进面板所属页）触发数据刷新
                            →    6. 失败处理（错误码见 00 文档 §8）：
                                   a. 403 AUTH_FORBIDDEN（TE 为非本人归属线索创建）：
                                      `TDToast` "无权为该线索创建日程"
                                   b. 400 VALIDATION（参数错误/字段超限/
                                      userId 不属于本租户）：
                                      `TDToast` 显示后端 error.message
                                   c. 404 NOT_FOUND（线索/通话记录不存在）：
                                      `TDToast` "关联线索不存在"
                                   d. 500（服务端错误）：
                                      `TDToast` "创建失败，请稍后重试"
                                   e. 网络错误：
                                      `TDToast` "网络异常，请重试"
                                   f. 所有失败情况：按钮恢复可用，loading 消失
```

## 6. 接口依赖

### 6.1 创建日程（主接口）

```
POST /api/tenant/schedules
```

**请求 Body：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `leadId` | string | 是 | 关联线索 ID（UUID 字符串），从入口参数获取 |
| `scheduledAt` | int（Unix 秒） | 是 | 计划时间（UTC Unix 秒；展示层 `YYYY-MM-DD HH:mm` → Unix 秒换算见 00 文档 §7） |
| `title` | string | 否 | 日程标题，最大 200 字符 |
| `content` | string | 否 | 日程内容，最大 2000 字符 |
| `userId` | string | 否 | 归属人 ID，仅 TM/TA 可传（须属于本租户，否则 400）；不传时默认为当前用户 |
| `callRecordId` | string | 否 | 关联通话记录 ID（从跟进面板入口可能传入；后端校验存在性） |

**请求示例：**

```json
{
  "leadId": "lead_001",
  "scheduledAt": 1784098800,
  "title": "跟进通话结果",
  "content": "客户对三居室感兴趣，需安排看房",
  "userId": "user_001"
}
```

> `1784098800` = 2026-07-15 15:00（北京时间 +08:00）。

**成功响应（HTTP 201）：**

```json
{
  "success": true,
  "data": {
    "id": "sch_002",
    "leadId": "lead_001"
  },
  "error": null
}
```

> 后端创建成功仅返回 `{id, leadId}`，不返回完整日程对象；如需完整数据由上一页自行刷新。

**错误响应**（错误码定义见 00 文档 §8）：

| HTTP 状态码 | error.code | 说明 | 前端处理 |
|------------|------------|------|----------|
| 400 | `VALIDATION` | 参数校验失败（字段超限 / `callRecordId` 不存在 / `userId` 不属于本租户） | `TDToast` 显示 `error.message` |
| 403 | `AUTH_FORBIDDEN` | TE 为非本人归属线索创建日程 | `TDToast` "无权为该线索创建日程" |
| 404 | `NOT_FOUND` | 线索不存在 | `TDToast` "关联线索不存在" |
| 500 | - | 服务端异常 | `TDToast` "创建失败，请稍后重试" |

### 6.2 获取线索摘要

```
GET /api/tenant/leads/:leadId
```

用于页面初始化时获取关联线索的姓名、电话、项目信息。

**响应结构（仅需要的字段）：**

```json
{
  "success": true,
  "data": {
    "id": "lead_001",
    "name": "张伟",
    "phone": "13812341234",
    "project": "XX花园"
  },
  "error": null
}
```

### 6.3 获取用户下拉列表（TM/TA 选归属人）

```
GET /api/tenant/options/users
```

用于归属人下拉选择器。⚠️ 不使用 `GET /api/tenant/users?role=te`——`/users` 挂 `requireTA` 中间件（TM 调用返回 403 `AUTH_FORBIDDEN`），且 `options/users` 不支持 `role` 过滤参数，返回全租户用户（含 TM/TE）。

**响应结构：**

```json
{
  "success": true,
  "data": [
    { "id": "user_001", "name": "赵六", "role": "TE" },
    { "id": "user_002", "name": "钱七", "role": "TE" },
    { "id": "user_003", "name": "孙八", "role": "TM" }
  ],
  "error": null
}
```

### 6.4 联动说明

| 联动 | 说明 |
|------|------|
| 创建日程后 → 更新线索 `nextFollowupAt` | 后端自动完成，前端无需额外请求。创建日程成功后，关联线索的 `nextFollowupAt` 字段会被后端自动更新为日程的 `scheduledAt` |
| 线索详情页刷新 | 新建日程页面关闭后，线索详情页 `onResume` 时重新请求线索详情，以展示更新后的 `nextFollowupAt` |

## 7. 边界情况与备注

### 7.1 边界情况

| 场景 | 处理方式 |
|------|----------|
| leadId 对应的线索不存在 | 请求线索摘要返回 404 → 显示错误态"线索不存在" + `TDButton` 返回按钮 |
| leadId 对应的线索不属于当前用户（TE） | 页面正常显示，但创建按钮禁用 + 底部 `error-7` 红色提示"该线索不属于您，无法创建日程" |
| 入口 1 默认时间已过期（用户停留过久） | 正常显示，提交时 `TDToast` 提示"时间已过去，日程将标记为已逾期"，允许继续 |
| 快捷 chip 时间因跨天变为已过去 | 同上，提交时提示已逾期；chip 文案不变（仍为"明天/后天"，按创建页面打开时计算） |
| 计划时间未填写直接提交（入口 2 常见） | 输入框标红 + 下方提示"请选择计划时间"，阻止提交 |
| 网络断开时提交 | `TDToast` "网络异常，请重试"，按钮恢复可用 |
| 快速连续点击创建按钮 | 按钮立即进入 loading + 禁用状态，防止重复提交 |
| 标题/内容包含特殊字符（emoji、换行等） | 正常支持，URL encode 后传给后端 |
| 从跟进面板进入时 callRecordId 传入 | 作为可选参数一并传给创建接口 |
| 键盘弹起遮挡输入框 | 全屏页面由 `Scaffold` 的 `resizeToAvoidBottomInset: true`（默认）自动压缩布局，`ScrollView` 保证当前输入框滚动到可见区域 |
| 页面旋转（横竖屏切换） | 表单数据通过 StateManagement 保存，旋转后恢复 |
| APP 被系统回收后恢复 | 表单数据从保存的状态管理器恢复 |
| TM/TA 创建日程时未选择归属人 | 默认使用当前用户 ID，无需强制选择 |
| 归属人列表为空 | 归属人选择器显示"暂无可选员工"，禁用选择 |

### 7.2 表单校验规则速查

| 字段 | 是否必填 | 校验规则 | 错误提示 |
|------|---------|---------|----------|
| leadId | 是（自动） | 从入口参数获取，无需用户输入 | - |
| scheduledAt | 是 | 不能为空；提交时换算为 int Unix 秒（UTC） | "请选择计划时间" |
| title | 否 | 最大 200 字符 | "标题不能超过200个字符" |
| content | 否 | 最大 2000 字符 | "内容不能超过2000个字符" |
| userId | 否（TM/TA） | 若显示则必须有值（有默认值） | - |

### 7.3 备注

| 项目 | 说明 |
|------|------|
| 页面形式 | 两个入口均为**全屏 Push 页面**（右滑入）——产品决策已确认，不采用 `TDPopup` BottomSheet 形式 |
| 默认时间计算 | 仅入口 1：`当前时间 + 1 小时`，分钟取整到 5 的倍数（如 14:03 → 15:00，14:37 → 15:35）；入口 2 无默认值，需主动选择 |
| 数据保持 | 使用 StateManagement（如 GetX / Provider / Riverpod）管理表单状态，确保配置变更（旋转等）不丢失数据 |
| 暗色模式 | 所有颜色使用 TDesign Token，通过 `TDTheme.needMultiTheme()` 自动适配暗色模式 |
| 无障碍 | 所有输入框有 `label` 和 `semanticsLabel`；`TDCalendarPicker` / `TDDatePicker` 支持 TalkBack |
| 返回数据 | 页面关闭时，通过 StateManagement 或回调通知上一页刷新数据 |
