## 1. 页面概述

### 1.1 页面定位

跟进时间线是线索详情页的核心信息区域，以竖向时间线的形式展示该线索的所有跟进历史。每条记录对应一次通话跟进或手动备注，按时间倒序排列（最新在顶部）。时间线既是回顾工具，也是工作质量的可视化呈现。

### 1.2 目标用户

| 角色 | 使用场景 |
|------|---------|
| **TE（租户员工）** | 查看自己的跟进历史，回忆上次沟通内容，编辑/删除近期误操作 |
| **TM（租户经理）** | 查看团队成员的跟进记录，了解线索跟进进展 |
| **TA（租户管理员）** | 查看全部跟进记录，审计跟进质量 |

### 1.3 核心目标

- **信息一目了然**：时间、跟进人、通话结果、备注内容，一条记录完整呈现
- **快速定位**：倒序排列 + 分页加载，最近的跟进最先看到
- **可控可纠**：支持编辑和删除，允许纠正误操作

### 2.3 操作按钮（直接显示，不使用隐藏菜单）

> **设计决策**：不使用长按或 ⋮ PopupMenu 隐藏菜单，每条跟进卡片右上角**直接显示**编辑/删除图标按钮（与 05-线索详情页时间线的内联操作按钮一致，减少一次点击）。按钮按角色与时效条件显示/隐藏，规则见 §3.9。

```
    ┌────────────────────────────────────────┐
    │  14:30  ·  2分钟前             [✏️][🗑️]│  ← TDButton(text) 图标按钮 32×32px
    └────────────────────────────────────────┘

    显示规则（详见 §3.9）：
    - TE：仅自己的记录显示按钮；编辑按钮限创建后 5 分钟内，
      删除按钮对自己的记录始终显示（无 5 分钟限制）
    - TM / TA：所有记录均显示编辑 + 删除按钮（无时限限制）
    - 无权限时按钮不渲染（不置灰），避免误导
```

| 属性 | 值 |

## 4. 交互流程

### 4.1 主流程：查看跟进时间线

```
用户操作                          系统响应
─────────                        ─────────
1. 在线索列表点击某线索           
                                  → 进入线索详情页
                                  → 加载线索基本信息（§A 区）
                                  → 加载跟进时间线
                                  → 显示 TDSkeleton 骨架屏（时间线区域）
                                  
                                  → API: GET /api/tenant/leads/:id/followups?page=1&size=20
                                  → 返回成功：
                                    a) 隐藏骨架屏
                                    b) 渲染跟进记录列表（倒序）
                                    c) 每条记录显示时间、跟进人、类型、内容
                                    d) 底部显示 TDLoadMore 分页指示器
                                    e) 根据权限显示/隐藏编辑/删除按钮

2. 阅读跟进内容                 
   内容被截断（> 3 行）           
                                  → 显示 [展开全文] 按钮
   点击 [展开全文]               
                                  → 卡片高度动画展开（200ms Curves.easeOut）
                                  → 显示完整内容
                                  → 按钮变为 [收起]
   点击 [收起]                   
                                  → 卡片高度动画收起（200ms Curves.easeIn）
                                  → 恢复 3 行截断

3. 滚动查看更多记录             
   向下滚动                      
                                  → 时间线跟随父级滚动（Sliver 区块共享页面滚动）
   滚动到底部                    
                                  → 自动触发 TDLoadMore 加载下一页
                                  → 底部显示 loading 指示器
                                  → API: GET .../followups?page=2&size=20
                                  → 追加新记录到列表底部
                                  → 若返回数量 < size → 显示"没有更多了"
```

### 4.2 下拉刷新（由父页面统一控制）

> 时间线作为 Sliver 区块嵌入线索详情页，**不含独立的 TDRefreshHeader**；下拉刷新由父页面（05）统一触发，时间线随父页面刷新一并重载。

```
用户操作                          系统响应
─────────                        ─────────
1. 在线索详情页顶部下拉           
   （滚动位置位于页面顶部）       
                                  → 父页面 TDRefreshHeader 显示刷新指示器
                                  → 下拉距离 > 阈值 → 触发刷新
```

## 7. 边界情况与备注

### 7.1 数据边界

| 场景 | 处理 |
|------|------|
| 跟进记录数量 = 0 | 显示 TDEmpty 空态（§5.2） |
| 跟进记录数量 > 100 | TDLoadMore 分页加载，每次 20 条；滚动流畅无卡顿 |
| 跟进内容 = 空字符串 | 不显示内容区域，仅显示元信息行 |
| 跟进内容 > 2000 字 | 正常显示，截断 3 行 + 展开；展开后卡片内滚动 |
| 跟进内容含换行符 | 保留换行，正常渲染 |
| 跟进内容含 URL | 可点击，打开浏览器 |
| 跟进人已被删除 | 显示 "已离职员工" 或 "未知"，userName 字段由后端兜底 |
| 通话时长 = 0 且 answerType = answered | 显示 "已接听 · 0秒"（可能是接听后立即挂断） |
| 分类变更但无旧分类名 | 仅显示 "分类变更为 {新分类}" |

### 7.2 权限边界

| 场景 | 处理 |
|------|------|
| TE 尝试编辑他人记录 | 编辑按钮不显示；若通过接口调用 → 403 |
| TE 编辑超 5 分钟 | 编辑按钮不显示；若通过接口调用 → 403 + "编辑时限已过" |
| TE 删除他人记录 | 删除按钮不显示；若通过接口调用 → 403 |
| TM/TA 编辑/删除任何记录 | 所有按钮可见，接口正常响应 |
| 线索被转移给他人 | TE 失去查看权限 → 详情页返回 "无权访问" → 返回列表 |

### 7.3 性能优化

| 项目 | 要求 |
|------|------|
| 列表渲染 | 使用 `SliverList`（delegate 按需构建子项）+ `AutomaticKeepAliveClientMixin`，避免全量刷新 |
| 图片加载 | 跟进记录无图片，无需图片加载库 |
| 骨架屏 | 使用 `TDSkeleton` 组件 |
| 分页预加载 | 滚动到倒数第 3 条时预加载下一页 |
| 时间更新 | 相对时间每 60s 刷新一次（仅更新可见项） |
| 内存 | 列表项复用；单条记录内存 < 50KB |

### 7.4 动画规格

| 动画 | 时长 | 曲线 |
|------|------|------|
| TDSkeleton → 真实内容 | 300ms | fade in (`Curves.easeInOut`) |
| 卡片展开/收起 | 200ms | `Curves.easeInOut` |
| 新记录淡入 | 300ms | fade in |
| 新记录高亮消失 | 2000ms → 500ms fade | linear hold + fade out |
| 删除滑出 | 300ms | `Curves.easeIn` |
| 列表重排 | 200ms | 隐式动画 |
| TDRefreshHeader 下拉刷新指示器 | 跟随手指 | TDesign 自定义刷新头（品牌色旋转动画） |

### 7.5 无障碍

| 项目 | 要求 |
|------|------|
| 时间线圆点 | semanticsLabel = "已接听跟进" / "无人接听跟进" 等 |
| 跟进卡片 | semanticsLabel 包含：时间 + 跟进人 + 类型 + 内容摘要 |
| 编辑按钮 | semanticsLabel = "编辑跟进备注" |
| 删除按钮 | semanticsLabel = "删除跟进记录" |
| 展开全文 | semanticsLabel = "展开完整内容" |
| TDEmpty 空态 | semanticsLabel = "暂无跟进记录" |
| 焦点顺序 | 从上到下，从左到右（时间线自然阅读顺序） |

### 7.6 暗色主题适配

| 组件 | 亮色 | 暗色 |
|------|------|------|
| 卡片背景 | `gray-13`（#FFFFFF） | `gray-11`（#2C2C2C） |
| 卡片边框 | `gray-2`（#EEEEEE） | `gray-10`（#3C3C3C） |
| 时间文字 | `gray-12`（#181818） | `gray-1`（#F3F3F3） |
| 相对时间 | `gray-6`（#A6A6A6） | `gray-6`（#A6A6A6） |
| 跟进内容 | `gray-12`（#181818） | `gray-1`（#F3F3F3） |
