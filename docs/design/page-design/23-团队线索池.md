## 1. 页面概述

### 1.1 页面定位

**团队线索池不是独立页面**，它就是**「线索」Tab（Tab 2）的线索列表页本身在 TM/TA 角色下的默认视图**：TM/TA 进入线索 Tab 时，页面自动以 `scope=all` 拉取团队全量线索（TE 进入同一页面则自动 `scope=mine` 只看自己）。与 TE 视角相比，TM/TA 视角额外支持按归属人筛选、每条线索显示归属人。页面为**受限操作模式**：可查看、可代录跟进、可拨号，但不提供领取/分配/回收等操作入口（领取为 TE 专属；分配/回收在后台 Web 完成）。

> **v1.1 重新定位说明**：原"独立页面（个人中心 → 团队管理进入）"方案已随 [20-团队入口](./20-团队入口.md) 归档废弃，遵循主文档（`APP_PRODUCT_DESIGN.md` §4.8.1）"TM/TA 在「线索」Tab 自动展示全部线索"的架构。**本页与 [04-线索列表](./04-线索列表.md) 描述的是同一页面**，本文档仅描述 TM/TA 视角的差异部分（多一个归属人筛选器、`scope=all`、受限操作模式），共用组件规格（搜索框、筛选面板、线索卡片、无限滚动等）一律引用 04 文档，不重复定义。

### 1.2 目标用户

| 角色 | 团队视图是否可见 | 说明 |
|------|:--------:|------|
| TE（租户员工） | ❌ | 线索 Tab 对 TE 自动 `scope=mine`（仅自己的线索），不显示归属人筛选器 |
| TM（租户经理） | ✅ | 线索 Tab 自动 `scope=all`，查看全团队线索（受限操作模式） |
| TA（租户管理员） | ✅ | 线索 Tab 自动 `scope=all`，查看全团队线索（受限操作模式） |

### 1.3 核心目标

- 浏览团队全部线索，了解整体线索状况
- 按归属人筛选，查看特定员工的线索情况
- 快速搜索定位特定线索
- 点击线索进入详情（受限操作模式），可查看跟进历史、发起拨号、代录跟进
- 顶部统计摘要条实时反映线索状态分布

### 1.4 入口路径

```
线索 Tab（Tab 2）→ TM/TA 登录后自动展示团队全量线索（scope=all&erased=0）→ 即"团队线索池"视图
```

> 无独立入口、无额外切换操作：后端按角色自动决定数据范围（TE=`mine`，TM/TA=`all`），APP 无需传 `scope` 参数也会得到正确范围，但为语义明确建议 TM/TA 视角显式携带 `scope=all`。

### 1.5 同一页面的两种角色视角差异

> 下表为**线索 Tab 同一页面**在 TE 视角与 TM/TA 视角下的差异（v1.1 整改：不再是"两个页面"的对比）。

| 特性 | TE 视角 | TM/TA 视角（本文档） |
|------|-----------|----------------|
| 数据范围 | `scope=mine`（仅自己的） | `scope=all`（全部） |
| 归属人显示 | 不显示（全是自己的） | 每条线索显示归属人姓名 |
| 归属人筛选 | 无 | 可按归属人筛选 |
| 操作权限 | 可编辑自己线索的信息 | **受限操作模式**：可查看他人线索，不可编辑他人线索基础信息 |
| 拨号 | 可拨号 | 可拨号（TM/TA 也有拨号权限） |
| 跟进记录 | 可查看 + 新增 | 可查看 + **可代录跟进**（后端允许 TM/TA 为任意线索添加跟进） |
| 领取公海 | 可领取（`claim`，需租户开启 `allowSelfClaim`） | **不可领取**——`POST /leads/:id/claim` 仅 TE 可调，TM/TA 调用返回 403 |
| 分配/回收 | 无 | APP 内无操作入口（后端 Web 操作；后端接口 `assign`/`recycle` 为 `requireManager`） |

## 3. 组件规格

### 3.1 页头（沿用线索 Tab 页头，规格见 04）

> **v1.1 整改**：团队线索池视图无独立 `TDNavBar`（不是 Push 页面，无返回箭头）。页头与 [04-线索列表](./04-线索列表.md) 完全一致（标题/搜索入口等以 04 为准），TM/TA 视角仅在线索列表区域增加归属人筛选能力。

| 属性 | 值 |
|------|-----|
| 类型 | 线索 Tab 内页头（无导航图标、无返回行为） |
| 底部 TabBar | 始终保留 `TDBottomTabBar`（首页/线索/日程/我的） |

### 3.2 统计摘要条

| 属性 | 值 |
|------|-----|
| 类型 | 自定义容器 |
| 高度 | 40px（固定） |
| 背景色 | `gray-1`（`#F3F3F3`） |
| 内边距 | 左右 `spacer16`（16px） |
| 布局 | 水平排列（Row），各项间隔 `spacer16`（16px） |

**内容布局**：

```
┌──────────────────────────────────┐
│ 共 1,234 条  跟进中 345  待跟进 128│
└──────────────────────────────────┘
```

| 子元素 | 规格 |
|--------|------|
| 总线索 | "共 **1,234** 条"，`fontBodySmall`，12sp，`gray-12`；数字 `bold`，`brand-7` 色 |
| 跟进中 | "跟进中 **345**"，`fontBodySmall`，12sp；数字 `bold`，`success-7` 色（`#2BA471`） |
| 待跟进 | "待跟进 **128**"，`fontBodySmall`，12sp；数字 `bold`，`warning-7` 色（`#E37318` 橙色） |
| 分隔 | 各项之间用竖线 `|` 分隔，`gray-2` 色 |

**数据来源（v1.1 明确）**：

| 摘要项 | 数据来源 | 说明 |
|--------|---------|------|
| 共 X 条 | 列表接口 `GET /api/tenant/leads` 响应分页体 `data.total` | 随筛选/搜索条件变化，**MVP 即可落地** |
| 跟进中 Y / 待跟进 Z | **暂无对应后端字段** | 列表接口不返回按状态聚合的统计；`GET /api/tenant/stats` 虽有 `byStatus`，但为**按 dateFrom/dateTo 日期范围**的统计口径（且必传日期参数），与"当前全量快照"语义不符 |

> **MVP 处理**：摘要条**仅显示"共 X 条"**（来自列表 `total`），"跟进中/待跟进"两项待后端补充实时聚合统计能力后再上线——**v1.1 依赖后端统计能力，MVP 可仅显示 total**。摘要条布局预留位置，能力上线前该两项不渲染（不显示占位 0）。

| 文案格式 | "下次跟进: {时间} 🔔" |
| 今天 | 🔔 图标 + 文字 `brand-7` 色（提醒关注） |
| 已过期 | 🔔 图标 + 文字 `error-7` 色 |
| 未来 | 🔔 图标 + 文字 `gray-6` |
| 无下次跟进 | 不显示此行 |
| 位置 | 跟进摘要下方，间距 `spacer4`（4px） |

### 3.8 排序切换

| 排序选项 | API 参数 | 说明 |
|---------|---------|------|
| 最近更新（默认） | `sort=-updatedAt` | 按最后更新时间降序（`-` 前缀为降序，无独立 `order` 参数） |
| 下次跟进时间 | `sort=nextFollowupAt` | 待跟进优先，升序排列（无 `-` 前缀即升序） |

排序按钮在两个选项间循环切换：

```
点击排序按钮
  │
  ├─ 当前"最近更新" → 切换为"下次跟进时间"
  │   └─ 按钮文案变为"下次跟进"
  │
  └─ 当前"下次跟进" → 切换为"最近更新"
      └─ 按钮文案变为"最近更新"
```

## 6. 接口依赖

> **全局约定（v1.1 整改）**：本章接口格式以 [00-全局API约定](./00-全局API约定.md) 为唯一事实来源。响应统一信封为 `{ "success": true, "data": { ... }, "error": null }`；列表接口 `data` 为分页体 `{ items, total, page, size, pages }`（请求参数 `page`/`size`，不使用 offset/limit）。过滤参数用**驼峰字段名**（裸字段即等值匹配），排序用 `sort=-字段名`（`-` 前缀为降序）。

### 6.1 核心接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 团队线索列表 | GET | `/api/tenant/leads` | TM/TA 调用 `scope=all` 返回全团队线索 |
| 成员列表（归属人筛选） | GET | `/api/tenant/options/users` | 轻量下拉接口（id+name+role），全角色可读。**不可用 `/api/tenant/users`**（挂 `requireTA`，TM 调用返回 403 `AUTH_FORBIDDEN`） |

### 6.2 线索列表请求参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:----:|--------|------|
| `scope` | string | ✅ | — | 固定传 `all`（TM/TA 视角） |
| `erased` | int | ✅ | — | 固定传 `0`（排除已擦除记录） |
| `page` | int | 否 | 1 | 页码（从 1 起） |
| `size` | int | 否 | 20 | 每页条数（最大 200） |
| `q` | string | 否 | — | 搜索关键词（姓名/电话/公司） |
| `ownerId` | string (UUID) | 否 | — | 按归属人筛选（驼峰字段名，裸字段即等值匹配） |
| `status` | string | 否 | — | 按状态筛选（`pending`/`assigned`/`following`/`converted`/`invalid`） |
| `categoryId` | string | 否 | — | 按分类筛选 |
| `sort` | string | 否 | `-updatedAt` | 排序：`-` 前缀为降序（如 `-updatedAt`），无前缀为升序（如 `nextFollowupAt`）。**不使用独立 `order` 参数** |

**分页响应体**（`data` 内）：

```json
{ "items": [ /* 线索对象，见 §6.3 */ ], "total": 1234, "page": 1, "size": 20, "pages": 62 }
```

### 6.3 响应字段映射（线索对象）

| 响应字段 | 页面使用位置 | 格式化 |
|---------|------------|--------|
| `id` | 点击跳转详情时传递 | — |
| `name` | 线索卡片 - 姓名 | 脱敏显示（如"王**"），后端返回已脱敏数据 |
| `phone` | 线索卡片 - 电话 | 脱敏显示（如"138****1234"），后端返回已脱敏数据 |
| `company` | 线索详情 | — |
| `status` | 线索卡片 - `TDTag` 状态标签 | 映射为中文 + 对应颜色 |
| `categoryId` | 线索卡片 - `TDTag` 分类标签 | 需映射为分类名称（从缓存的分类列表中查找） |
| `categoryName` | 线索卡片 - `TDTag` 分类标签 | 若后端直接返回分类名称则直接使用 |
| `ownerId` | 归属人颜色映射 key | — |
| `ownerName` | 线索卡片 - 归属人姓名 | 原文 |
| `nextFollowupAt` | 线索卡片 - 下次跟进时间 | 格式化为"今天"/"明天"/"X月X日"；判断是否过期 |
| `lastFollowupAt` | 线索卡片 - 上次跟进时间 | 格式化为"今天"/"昨天"/"X天前" |
| `lastFollowupSummary` | 线索卡片 - 跟进摘要 | 截取前 20 字 + 接听类型标签 |
| `updatedAt` | 排序依据 | 不直接展示 |
| `projectId` | 线索详情 | — |
| `isBlocked` | 不展示（已屏蔽的不返回） | — |

> **脱敏说明**：线索列表中的 `name` 和 `phone` 由后端进行脱敏处理后返回。APP 直接展示后端返回的脱敏值，前端不做额外脱敏处理。

### 6.4 请求时机

| 时机 | 请求 |
|------|------|
| 首次进入线索 Tab（TM/TA） | `GET /api/tenant/leads?scope=all&erased=0&page=1&size=20&sort=-updatedAt` + `GET /api/tenant/options/users`（若未缓存） |
| 搜索输入（防抖 500ms） | `GET /api/tenant/leads?scope=all&erased=0&q={keyword}&...`（重置 page=1） |
| 切换归属人筛选 | `GET /api/tenant/leads?scope=all&erased=0&ownerId={id}&...`（重置 page=1） |
| 切换状态筛选 | `GET /api/tenant/leads?scope=all&erased=0&status={status}&...`（重置 page=1） |
| 切换排序 | `GET /api/tenant/leads?scope=all&erased=0&sort={-updatedAt 或 nextFollowupAt}&...`（重置 page=1） |
| 下拉刷新 | 同首次进入（重置 page=1） |
| 无限滚动 | `GET /api/tenant/leads?scope=all&erased=0&page=N&size=20&...`（保持当前筛选条件） |

### 6.5 错误码处理

| HTTP 状态码 | 处理方式 |
|------------|---------|
| 200 | 正常渲染 |
| 400 | `TDToast` "请求参数有误" |
| 401 / 403 | 跳转登录页 |
| 429 | `TDToast` "请求过于频繁，请稍后再试" |
| 500 | 显示错误态 + 重试按钮 |
| 网络超时 | 显示错误态 + 重试按钮 |

## 7. 边界情况与备注

### 7.1 边界情况

| 场景 | 处理方式 |
|------|---------|
| 线索已被擦除 | `erased=0` 参数排除，不会出现在列表中 |
| 线索归属人被删除 | `ownerName` 可能为 null 或"已删除用户"，显示灰色"未知"标签 |
| 搜索关键词包含特殊字符 | 前端 URL 编码后传给后端，后端负责安全处理 |
| 筛选条件组合导致无结果 | 显示筛选后空态（见 5.2），提供"清除筛选"按钮 |
| 列表数据量极大（> 10000） | 后端分页，前端 `ListView` 懒加载，无性能问题 |
| 快速连续切换筛选条件 | 使用请求取消机制，只保留最后一次请求的结果 |
| 搜索输入极快 | 500ms 防抖 + 请求取消，避免无效请求 |
| 排序字段 `nextFollowupAt` 为 null | 排到列表末尾（null 值排最后） |
| 分类名称获取失败 | 显示分类 ID 前 4 位作为降级显示 |
| 线索电话为空 | 不显示电话部分，仅显示姓名 |
| 线索姓名和电话都为空 | 卡片仍显示，姓名位置显示"未命名线索"，电话位置不显示 |
| 与线索 Tab 的关系 | 本页即线索 Tab（Tab 2）页面本身在 TM/TA 角色下的视图（自动 `scope=all`），与 TE 视角共用同一页面与组件，仅多一个归属人筛选器（见 §1.1 / §1.5） |
| 网络恢复后自动重试 | 不自动重试，用户需手动下拉刷新或点击重试按钮 |

### 7.2 受限操作模式详细说明

TM/TA 视角下的线索详情页为**受限操作模式**（v1.1 整改：原称"只读模式"，但能拨号 + 能代录跟进 ≠ 只读，故更名）：

| 功能 | 可用性 | 说明 |
|------|:------:|------|
| 查看基础信息 | ✅ | 姓名、电话、公司、分类、项目等 |
| 查看跟进历史 | ✅ | 完整时间线 |
| 发起拨号 | ✅ | 调系统拨号盘，通话后回传 |
| 代录跟进 | ✅ | 后端允许 TM/TA 为任意线索添加跟进（`requireOwned` 仅限制 TE 操作他人线索） |
| 领取线索 | ❌ | `POST /api/tenant/leads/:id/claim` **仅 TE 可调**（领取公海无主线索），TM/TA 调用返回 403 `AUTH_FORBIDDEN` |
| 编辑基础信息 | ❌ | 他人线索无编辑按钮 |
| 删除线索 | ❌ | 无删除入口 |
| 分配/回收 | ❌ | APP 无操作入口（在后台 Web 操作；后端 `assign`/`recycle` 接口挂 `requireManager`，仅 TM/TA 可调） |
| 查看通话记录 | ✅ | 可查看该线索的所有通话记录 |

> **设计考量**：TM/TA 查看团队线索池的核心目的是"了解情况"而非"操作数据"。但拨号和代录跟进是日常协作的一部分（如经理帮忙跟进），因此保留这两项操作权限；领取是 TE 专属的公海认领动作，分配/回收属于管理动作、放在后台 Web 完成。

### 7.3 性能优化

| 策略 | 说明 |
|------|------|
| 请求取消 | 筛选条件变更时取消上一次未完成的请求 |
| 搜索防抖 | 输入停止 500ms 后才触发搜索请求 |
| 图片缓存 | 列表无图片资源，不涉及 |
| 成员列表缓存 | `GET /api/tenant/options/users` 结果首次加载后缓存至内存，后续不重新请求 |
| 分类映射缓存 | 分类 ID → 名称映射从首次使用时缓存 |
| ListView 懒加载 | 使用 `ListView` 懒加载列表项，离屏不渲染 |
