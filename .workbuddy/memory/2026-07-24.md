## 定位「手机登不上 / 接口无响应」= 手机没走 Mac 代理（非代码问题）
- 现象：app 登录/所有接口「有请求、无响应」；Mac 上 curl 一切正常。
- 根因（已实测坐实）：Mac 通过本机 HTTP 代理访问测试服务器——`env` 含 `HTTPS_PROXY=http://127.0.0.1:52702`（进程 `sandbox-c`，仅绑 Mac 回环）；`curl` 读该变量故通，`ping` 走 ICMP 不经代理故也失败。
  手机无此代理，且 `tm-api-test.kao9.com` 在手机 DNS 解析到 `198.18.8.151`（RFC2544 TEST-NET-2 不可路由黑洞地址）→ 100% 丢包 → 请求回不来。与 App 代码 / Alice 集成无关，零代码改动。
- 手机侧实测：`ping 8.8.8.8` 通（手机本身联网），`ping tm-api-test.kao9.com` 100% 丢包，解析到 198.18.8.151。
- 修复（已搭好通道，待用户在手机 UI 补一步）：
  1. `adb reverse tcp:52702 tcp:52702`（手机本机 52702 → Mac 本机 52702，已生效）；
  2. 手机 WiFi 设置 → 修改网络 → 高级 → 代理=手动 → 主机名 `127.0.0.1`、端口 `52702`（adb 设全局代理需 WRITE_SECURE_SETTINGS，非 root 不可，故走 WiFi 手动代理）。
- 注意：`adb reverse` 隧道仅 adb 连接期间有效；拔线/断开即失效。要永久可用，可改代理客户端监听 `0.0.0.0` 并把手机代理填 Mac 局域网 IP `192.168.8.88:52702`。撤销：`adb reverse --remove tcp:52702`。
- 通用排查套路（可复用）：手机接口无响应先 `adb shell ping <域名>` 与 `ping 8.8.8.8` 对比，再查 Mac `env | grep -i proxy` 与域名解析 IP 是否两端一致。
