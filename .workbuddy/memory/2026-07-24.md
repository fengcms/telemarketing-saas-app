## 定位「手机登不上 / 接口无响应」= 手机没走 Mac 代理（非代码问题）
- 现象：app 登录/所有接口「有请求、无响应」；Mac 上 curl 一切正常。
- 根因（已实测坐实）：Mac 通过本机 HTTP 代理访问测试服务器——`env` 含 `HTTPS_PROXY=http://127.0.0.1:52702`（进程 `sandbox-c`，仅绑 Mac 回环）；`curl` 读该变量故通，`ping` 走 ICMP 不经代理故也失败。
  手机无此代理，且 `tm-api-test.kao9.com` 在手机 DNS 解析到 `198.18.8.151`（RFC2544 TEST-NET-2 不可路由黑洞地址）→ 100% 丢包 → 请求回不来。与 App 代码 / Alice 集成无关，零代码改动。
- 手机侧实测：`ping 8.8.8.8` 通（手机本身联网），`ping tm-api-test.kao9.com` 100% 丢包，解析到 198.18.8.151。
- 修复（已搭好通道，待用户在手机 UI 补一步）：
  1. `adb reverse tcp:52702 tcp:52702`（手机本机 52702 → Mac 本机 52702，已生效）；
  2. 手机 WiFi 设置 → 修改网络 → 高级 → 代理=手动 → 主机名 `127.0.0.1`、端口 `52702`（adb 设全局代理需 WRITE_SECURE_SETTINGS，非 root 不可，故走 WiFi 手动代理）。
- 注意：`adb reverse` 隧道仅 adb 连接期间有效；拔线/断开即失效。要永久可用，可改代理客户端监听 `0.0.0.0` 并把手机代理填 Mac 局域网 IP `192.168.8.88:52702`。撤销：`adb reverse --remove tcp:52702`。
- 通用排查套路（可复用）：手机接口无响应先 `adb shell ping <域名>` 与 `ping 8.8.8.8` 对比，再查 Mac `env | grep -i proxy` 与域名解析 IP 是否两端一致。

## 【真因·用户定位】release 全版本连不上 = main 清单缺 INTERNET 权限（推翻上一段"Alice 顶层 final"判断）

## 线索详情「新建日程」日期/时间选择器「确定」无反应（14:18 起）
- 现象：线索详情→新建日程抽屉里，点日期/时间选择控件的左侧「取消」可关，右侧「确定」点了无反应；date/time 两控件都如此。
- 根因（翻 tdesign_flutter 0.2.7 源码坐实）：`TDPicker.showDatePicker` 的 `onConfirm` 类型是 `typedef DatePickerCallback = void Function(Map<String, int> selected)`——回调拿到的是 **Map**（键 year/month/day/hour/minute/second，未启用列值为 -1），**不是 DateTime**。而 `lib/pages/leads/widgets/schedule_dialog.dart` 两处 onConfirm 都写成 `final dt = date as DateTime;`：
  ① release 下该 cast 抛 `CastError` 被吞 → 确定"无反应"（debug 模式会红屏）；
  ② `TDDatePicker._buildHeader` 的「确定」只调 `widget.onConfirm!(selected)`、**不自动 pop**（取消分支因未传 onCancel 才走 else 里的 pop）。故即便修好类型，回调里也必须自己 `Navigator.of(context).pop()` 关掉选择器。
- 修复：`schedule_dialog.dart` 两处 onConfirm 改为 `final map = selected;`（selected 经 DatePickerCallback 推断已是 Map<String,int>，去掉多余 as 消 warning），从 map 取 year/month/day 或 hour/minute 重建 DateTime / TimeOfDay，并补 `Navigator.of(context).pop()`。对照 tdesign 官方示例 `td_date_picker_page.dart` 写法（也是读 map + 手动 pop）。`flutter analyze` 零问题。
- 教训（可复用）：用 tdesign_flutter 的 TDPicker，onConfirm/onChange 参数是 `Map<String,int>` 不是 DateTime；且确认后必须手动 pop 选择器。项目 STYLE 约定"不确定就翻三方源码/官方示例核实，不靠猜"又一次避免误判。
